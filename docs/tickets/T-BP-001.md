# T-BP-001: AppShell layout slot 化(挙動不変リファクタ)

- Status: done
- 担当: Sonnet subagent
- 依存: なし(ADR-0007)

## 背景

Bottom Panel / Status Bar / Header Toolbar を載せるため、KagiApp::render の構造を
「縦 flex のスロット」に再編する。**このチケットは見た目・挙動を一切変えない純リファクタ**
(スロットの器だけ作る。新 UI は後続チケット)。

## スコープ

1. `KagiApp::render` を次の構造に再編(ADR-0007):
   ```
   div(flex_col, size_full)
     .child(render_header_slot(..))    // 今は既存の header bar(repo名等)をそのまま返す
     .child(render_body(..))           // 既存の sidebar|list|panel 一式を関数抽出(中身は移動のみ)
     .child(render_bottom_panel_slot(..)) // 今は空(None を返す)。器だけ
     .child(render_status_bar(..))     // 既存 status footer をそのまま移す(将来の StatusBar の席)
   ```
2. 既存コードの**移動と関数抽出のみ**。ロジック・スタイル・id・イベント配線は変更禁止
3. `render_*_slot` 系の関数シグネチャは後続チケット(T-BP-002/003, T-HT-001)が拡張しやすい形に
   (`&mut KagiApp` 相当 + `&mut Context<KagiApp>` を受ける)

## 実装メモ

### 抽出した関数構成

`impl Render for KagiApp` の `render` メソッドは以下の構造に再編した。
旧インライン実装は **移動のみ** (ロジック・スタイル・id・イベント配線変更なし)。

```
KagiApp::render (impl Render)
  ├─ self.render_header_slot(header, is_dirty, cx)   // 旧インライン header bar をそのまま抽出
  ├─ self.render_body(row_count, selected, detail, …, cx)  // 旧インライン body_row をそのまま抽出
  ├─ .children(self.render_bottom_panel_slot())      // 空(None を返す器だけ)
  ├─ [modal overlays — render 内に残す]
  └─ self.render_status_bar(status_footer)           // 既存 render_status_footer へ委譲
```

slot 系 4 メソッドは `impl Render` ブロックではなく別の `impl KagiApp` ブロックに置いた
(Rust の trait impl は trait のメソッドしか持てないため)。

`divider_drag_move` リスナーはルート div に `.on_drag_move` で付けるため `render` 内に残した。
`render_body` はパラメータが多いため `#[allow(clippy::too_many_arguments)]` を付与。

| env | 結果 |
