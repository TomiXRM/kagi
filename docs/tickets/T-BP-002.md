# T-BP-002: BottomPanel open/close + 高さリサイズ

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T-BP-001(ADR-0007)

## 背景

T-BP-001 で作った `render_bottom_panel_slot` の器に、開閉とリサイズができる空パネルを実装する。
タブの中身(Operation Log / Terminal)は後続。

## スコープ

1. 状態(ADR-0007): `KagiApp` に
   - `bottom_panel_open: bool`(default false)
   - `bottom_panel_height: f32`(default 220.0、クランプ 80.0〜viewport の 60%)
   - `bottom_tab: BottomTab`(`enum BottomTab { OperationLog, Terminal }`、default OperationLog)
2. `render_bottom_panel_slot`:
   - open 時のみ描画: 上端に水平ディバイダ(4px、T023/T030 と同じ見た目・`cursor_row_resize`)+
     タブバー(OperationLog / Terminal のラベル。アクティブ強調。クリックで切替)+
     中身プレースホルダ(「(empty — T-BP-004/007)」のテキストでよい)
   - リサイズ: 既存 `DividerKind` に `BottomPanel` を追加し、divider_drag_move に
     `height = viewport_h - cursor_y - status_bar_h(22.0) - 2` のケースを追加(絶対座標方式)
3. 開閉トグル:
   - 暫定の開閉ボタンを status bar 右端に置く(▲/▼ アイコン的テキストで可。本格的な StatusBar は T-BP-003)
   - キーボード: `cmd-j` で toggle。gpui 0.2.2 の action + KeyBinding の正しい登録方法を
     registry の examples(action 系)で確認して実装(`cx.bind_keys` / `actions!` マクロ等)
4. 高さ・open 状態・active tab は reload() で保持(フィールドコピー方式に倣い、コピー対象に入れない)
5. headless 検証 env: `KAGI_BOTTOM_PANEL=1` で起動時 open + `[kagi] bottom-panel: open height=H tab=T` ログ
