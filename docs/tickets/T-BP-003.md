# T-BP-003: StatusBar 本実装(情報表示 + Terminal/Output アイコン + cmd-j 補完)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T-BP-002(ADR-0007 / requirements-shell.md §2)
- 追加要望: ユーザー「フッターにターミナルのアイコンを置いて、クリックで Terminal が出てほしい」

## スコープ

1. 既存の status footer を **StatusBar** に拡張(高さ 22px のまま、左 → 右):
   - repo 名 / branch 名(detached/unborn は明示)/ dirty マーク(●)
   - `staged N` `unstaged M`(0 のときは省略)
   - `↑A ↓B`(upstream あり時。なければ `no upstream`)
   - 最終 refresh 時刻(`HH:MM:SS`。reload / reload_external のたびに更新。T017 の epoch→時刻変換を再利用)
   - 直近操作メッセージ(既存 FooterStatus。スペースが余ったときだけ flex_1 で)
   - **右端: アイコン群** — `>_`(Terminal)/ `≡`(Operation Log)。クリックで Bottom Panel の該当タブを
     **開く or 同タブ表示中なら閉じる**(VSCode 方式)。アクティブタブのアイコンは強調色
   - 既存の ▲/▼ トグルはアイコン群に統合して廃止
2. データ: snapshot から StatusBar 用のサマリを KagiApp に保持(`status_summary` 構造体等。
   reload / reload_external で更新)。ahead/behind は HEAD branch の UpstreamInfo
3. detached / unborn の表示は ADR-0010 の文言(`detached HEAD` / `no commits yet`)
4. headless: `[kagi] statusbar: <branch> ↑A ↓B staged=N unstaged=M` を reload 時にログ(検証用)
5. cmd-j の補完: モーダルを閉じたとき root_focus へフォーカスを返す(全モーダルの cancel/confirm 経路。
   ユーザー報告済みの「モーダル後に cmd-j が効かない」予防)
