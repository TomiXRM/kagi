## スコープ

1. 既存の status footer を **StatusBar** に拡張(高さ 22px のまま、左 → 右):
   - repo 名 / branch 名(detached/unborn は明示)/ dirty マーク(●)
   - `staged N` `unstaged M`(0 のときは省略)
   - `↑A ↓B`(upstream あり時。なければ `no upstream`)
   - 最終 refresh 時刻(`HH:MM:SS`。reload / reload_external のたびに更新。T017 の epoch→時刻変換を再利用)
   - 直近操作メッセージ(既存 FooterStatus。スペースが余ったときだけ flex_1 で)
   - **右端: アイコン群** — `>_`(Terminal)/ `≡`(Operation Log)。クリックで Bottom Panel の該当タブを
     **開く or 同タブ表示中なら閉じる**(VSCode 方式)。アクティブタブのアイコンは強調色
   - 既存の ▲/▼ トグルはアイコン群に統合して廃止
2. データ: snapshot から StatusBar 用のサマリを KagiApp に保持(`status_summary` 構造体等。
   reload / reload_external で更新)。ahead/behind は HEAD branch の UpstreamInfo
3. detached / unborn の表示は ADR-0010 の文言(`detached HEAD` / `no commits yet`)
4. headless: `[kagi] statusbar: <branch> ↑A ↓B staged=N unstaged=M` を reload 時にログ(検証用)
5. cmd-j の補完: モーダルを閉じたとき root_focus へフォーカスを返す(全モーダルの cancel/confirm 経路。
   ユーザー報告済みの「モーダル後に cmd-j が効かない」予防)

## 実装メモ

### 変更ファイル

- `src/ui/mod.rs` — 主実装
- `src/ui/file_tree.rs` — `#[allow(dead_code)]` で既存警告を抑制

### StatusBar 最終レイアウト (22px, 左→右)

```
[branch] [● dirty] [+N staged] [~M unstaged] [↑A ↓B | no upstream] [HH:MM:SS] [operation msg (flex_1)] [>_] [≡]
```

- `branch`: `main` / `detached HEAD (abc12345)` / `no commits yet (main)`
- `●` (COLOR_WARNING): is_dirty の場合のみ
- `+N` (COLOR_SUCCESS): staged > 0 の場合のみ
- `~M` (COLOR_WARNING): unstaged > 0 の場合のみ
- `↑A ↓B` (TEXT_SUB): upstream あり / `no upstream` (TEXT_MUTED): no_upstream
- `HH:MM:SS` (TEXT_MUTED): last_refresh_secs → format_hms で HH:MM:SS 変換
- operation msg: FooterStatus の色でflex_1(overflow_hidden)
- `>_` icon: Terminal タブ切替(アクティブ=TEXT_MAIN, 非アクティブ=TEXT_MUTED)
- `≡` (U+2261) icon: Operation Log タブ切替(同上)
- 旧 ▲/▼ トグルは廃止、アイコン群に統合

### StatusBarSummary 構造体

`src/ui/mod.rs` 内 `StatusBarSummary` に:
- `branch`, `is_dirty`, `staged`, `unstaged`, `ahead/behind (Option)`, `no_upstream`, `last_refresh_secs`
- `from_snapshot(snap)`: RepoSnapshot から構築
- `log_headless()`: `[kagi] statusbar: <branch> ↑A ↓B staged=N unstaged=M` を出力
- `format_hms(epoch_secs)`: civil算術で HH:MM:SS 変換(crate 追加なし)

`KagiApp` に `status_summary: StatusBarSummary` フィールド追加。
`from_snapshot` / `reload` で更新 (headless ログ自動出力)。
