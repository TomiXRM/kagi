## スコープ

1. メモリ上の操作ログ(ADR-0007):
   - `KagiApp` に `op_entries: VecDeque<OpLogEntry>`(最大500、新しい順表示)
   - 起動時に `$KAGI_LOG_DIR or ~/.kagi/operations.jsonl` の**末尾100件**を読み込み(手書き JSON の逆: 行を簡易パース。
     パース失敗行はスキップ。serde 禁止のため `oplog.rs` に最小パーサを追加してよい — lib 側変更はこの1点のみ許可)
   - `record_op` / `record_headless_op` 経由のすべての操作で `op_entries` にも追記
2. Operation Log タブの表示:
   - 1行 = `HH:MM:SS  op  outcome要約`(Success=緑 / Failed=赤 / Refused=黄)
   - 行クリックで展開(before/after の StateSummary、Failed は error、Refused は blockers を表示)。
     展開状態は単一(直前の展開は閉じる)で可
   - リストは uniform_list(将来増えるため)。新しいものが上
   - 空のとき「No operations yet」
3. **失敗時自動オープン**(requirements-shell.md §1): `OpOutcome::Failed` 記録時に
   `bottom_panel_open = true` + `bottom_tab = OperationLog` + cx.notify
4. headless: `KAGI_BOTTOM_PANEL=1` 時に `[kagi] oplog-tab: N entries` を起動ログに出す

### パーサの方式

`entry_to_json` が生成する形式専用の最小手書きパーサ。serde 不使用。
- `extract_str_field` — `"key":"value"` または `"key":number` をスキャン
- `extract_object_field` — ネストオブジェクト `{...}` をブレースカウントで抽出
- `extract_string_array` — `["str1","str2",...]` を文字列配列として抽出
- `unescape_json_str` — `\" \\ \n \r \t \uXXXX` のエスケープ復元
- 解析失敗行はスキップ(fail-safe)
