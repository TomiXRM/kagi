- Status: in-progress

## 背景

Bottom Panel の Terminal タブに本物のシェルを入れる(ADR-0008: gpui-terminal 0.1.0 + portable-pty 0.9 採用)。
repo root を cwd に、ユーザーのデフォルトシェルで起動する。

## スコープ

1. **依存追加(このチケットのみ許可)**: `gpui-terminal = "0.1.0"` / `portable-pty = "0.9"`
   - **一次資料は gpui-terminal の README / リポジトリの examples**(https://github.com/zortax/gpui-terminal —
     portable-pty との接続例があるはず)と docs.rs/gpui-terminal/0.1.0。
     `TerminalView::new(pty_writer, pty_reader, config, cx)` 形式の API
2. session 管理(ADR-0008):
   - **Terminal タブを最初に表示したとき**に PTY + shell を起動(lazy)。
     shell = `$SHELL`(なければ `/bin/zsh`)、cwd = repo root、引数なしの login/interactive 形(`-l` 等は gpui-terminal 例に倣う)
   - `KagiApp` に session(`Entity<TerminalView>` 等)を保持し、**タブを閉じても破棄しない**(明示 kill なし = アプリ終了まで保持)
   - shell 終了(exit callback)時は「(terminal exited — 再表示で再起動)」表示にし、次回表示で再起動
   - 起動失敗時は Operation Log に Failed として記録(op="terminal-start")+ パネルに失敗理由表示
3. 表示: Terminal タブの body に TerminalView を `flex_1 + min_h(0)` で配置。リサイズ(パネル高さ変更)に追従
   (gpui-terminal の resize callback / API を確認して配線)
4. フォーカス: Terminal タブ表示時に terminal にフォーカス(タイプ入力が通る)。
   **cmd-j は terminal フォーカス中も効くこと**(binding context を確認。効かない場合は既知制限としてメモ)
5. terminal 内 git 操作の反映は T029 watcher が既に担う(動作確認のみ: terminal なしの外部 git で代替確認可)
6. headless: `KAGI_BOTTOM_PANEL=1 KAGI_TERMINAL=1` で Terminal タブ起動 + `[kagi] terminal: started shell=<shell>` ログ。
   PTY が headless で動かない場合はその旨ログ(クラッシュしない)

## 実装メモ

### gpui-terminal 0.1.0 実 API (2026-06-12 調査)

**TerminalView::new シグネチャ**:
```rust
pub fn new<W: Write + Send + 'static, R: Read + Send + 'static>(
    stdin_writer: W,
    stdout_reader: R,
    config: TerminalConfig,
    cx: &mut Context<Self>,
) -> Self
```
ジェネリック (Box 不要)。内部で `Box<dyn Write+Send>` に消去される。

**コールバック型エイリアス** (view.rs より):
```rust
pub type ResizeCallback = Box<dyn Fn(usize, usize) + Send + Sync>;
pub type ExitCallback   = Box<dyn Fn(&mut Window, &mut Context<TerminalView>)>;
```
`with_resize_callback` / `with_exit_callback` は `impl Fn...` を受けてビルダーで `Self` を返す。

**フォーカス**: `view_entity.read(cx).focus_handle().clone()` → `window.focus(&fh)` で Terminal フォーカス。

**TerminalView は `impl Render`** を持つ。親 view から `div().child(view_entity.clone())` で埋め込み可能。

### session 管理実装

- `KagiApp` に `terminal_session: Option<terminal::KagiTerminalSession>` フィールドを追加
- `KagiTerminalSession` は `view: Option<Entity<TerminalView>>` + `start_error: Option<String>` を持つ
- **lazy 起動**: Terminal タブクリック時 (`tab_terminal_click` listener) と ステータスバーアイコン (`icon_terminal_click`) で `ensure_terminal()` を呼ぶ
- **KAGI_TERMINAL=1**: `run_app` 内の window 構築後に `ensure_terminal()` を呼ぶ
- **タブ閉じても保持**: `terminal_session` は `KagiApp` 上に存在し続ける。`view` を None にするコードは shell exit 時のみ
- **再起動**: `view == None` かつ Terminal タブ表示 → `ensure_terminal` が再起動 (現時点では exit_callback 未配線のため PTY 終了検出は PM 確認待ち)
- **起動失敗**: `record_op("terminal-start", Failed)` → Operation Log に記録 + auto-open

### PTY 接続 (portable-pty 0.9)

1. `NativePtySystem::default().openpty(PtySize { rows:24, cols:80, ... })`
2. `pair.slave.spawn_command(cmd)` でシェル起動 (slave はその後 drop)
3. `pair.master.take_writer()` → `TerminalView::new` の writer 引数
4. `pair.master.try_clone_reader()` → reader 引数
5. `Arc<Mutex<Box<dyn MasterPty+Send>>>` で master を保持し resize callback に渡す

### ハマった点

- `cx.new()` は `gpui::AppContext` トレイトのメソッドであり、`use gpui::AppContext;` が必要 (terminal.rs でのみ)
- `render_terminal_body` は cx を使わないため `_cx` に変更 (unused variable warning 対策)
- `is_running()` メソッドは未使用なので `#[allow(dead_code)]` を付与

### 既知制限 (ADR-0008 より)

- mouse selection / scrollback navigation: 未実装 (gpui-terminal 0.1.0 の既知制限)
- exit callback: 現時点では未配線。shell が exit しても view entity が残る。次フレームで描画が止まる。再起動ロジックは `view = None` → 次の `ensure_terminal` で機能するが、exit を検知するトリガーが未実装
- cmd-j: terminal フォーカス中は gpui-terminal が KeyDownEvent を消費する可能性あり。既知制限としてメモ
