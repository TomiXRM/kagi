# T-CM-002: 右クリック commit を selection state に反映

- Status: done
- 依存: T-CM-001
- 関連: docs/requirements-context-menu.md / ADR-0020〜0026

## スコープ

1. 右クリック時に対象 row を選択状態にする
2. 既存 `select()` はトグル仕様のため、選択済み行の再選択で解除されない**冪等ガード**を必ず入れる
   (jump_to_branch と同じパターン。2026-06-12 の回帰の教訓)
3. 選択に伴う detail / changed files 取得は既存の副作用に乗る

## リスク・規約

- select() のトグルと干渉しないこと(必須 unit 確認)
- force push / reset --hard / git clean をコードに追加するのは絶対禁止(ADR-0024 の隔離例外を除き、本チケットでは一切不可)
- 文字列切り詰めは chars() ベース(byte slice は日本語で panic)

## 実装メモ

- 右クリック選択は `if self.selected != Some(row_index) { self.select(row_index) }` の冪等ガードを通す。
- 既に選択済みの row を右クリックしても `select()` の toggle 解除は発生しない。
