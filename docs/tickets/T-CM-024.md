## リスク・規約

- シンボリックリンク・相対 path の正規化は canonicalize 依存にしすぎない(存在しない path で失敗するため手動正規化)
- force push / reset --hard / git clean をコードに追加するのは絶対禁止(ADR-0024 の隔離例外を除き、本チケットでは一切不可)
- 文字列切り詰めは chars() ベース(byte slice は日本語で panic)

## 実装メモ

- `validate_worktree_path(repo_root, input)` を追加。相対 path は repo root 基準で解決し、存在済み path / repo 内 path / 親 dir 不在を拒否。
- 存在しない target 自体は canonicalize せず、親 dir の canonical path と filename から normalized path を構築。
- 日本語 path を含む unit test を `tests/worktree_test.rs` に追加。
