# T-CM-061: 操作後 refresh の統合確認

- Status: todo
- 依存: T-CM-060
- 関連: docs/requirements-context-menu.md / ADR-0020〜0026

## スコープ

1. 各操作後に snapshot / graph / Inspector / status bar が更新されること
   (既存 reload() + watcher で担保されるはず — menu 経由で確認)
2. 漏れ(detached 化後の表示等)があれば補修

- [ ] 既存 headless 検証に回帰なし
- [ ] 実装メモを本ファイル末尾に追記

## テスト方法

1. `cargo test`(パイプで握りつぶさず exit code を確認)
2. `bash scripts/make_fixture.sh` の fixture / tempdir のみで検証(ユーザー repo 禁止)
3. UI 変更は PM がスクリーンショット確認(headless ログを併設すること)
