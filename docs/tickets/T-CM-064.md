## スコープ

1. fixture repo で headless 経由(KAGI_* env)の一連を回す:
   cherry-pick(成功 + conflict Refused)/ revert(成功)/ checkout commit(detached)/
   compare HEAD・WT
2. 各操作後の statusbar / oplog ログを期待値と突き合わせる検証スクリプト的手順を
   本チケットに記録(scripts/ は触らない)

1. `cargo test`(パイプで握りつぶさず exit code を確認)
2. `bash scripts/make_fixture.sh` の fixture / tempdir のみで検証(ユーザー repo 禁止)
3. UI 変更は PM がスクリーンショット確認(headless ログを併設すること)
