# T-HT-001: Header Toolbar UI skeleton + branch/ahead-behind 表示(T-HT-002 統合)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T-BP-001(ADR-0009 / ADR-0010 / requirements-shell.md §3)

## 背景

画面上部に Git 操作の入口を並べる。**このチケットはボタンの「見た目と disabled 判定」まで**。
各操作の plan/実行は T-HT-003 以降(既存実装がある Branch Create / Stash / 選択 commit への操作は配線してよい)。

## スコープ

1. `render_header_slot` を **2段構成**に:
   - 上段(新設 Toolbar、高さ 34px): 操作ボタン群 + 右側に branch/ahead-behind 表示
   - 下段(既存の repo 情報行はツールバーに統合して廃止 — repo 名は Toolbar 左端へ)
2. ボタン(左から): `Pull` `Push` | `Branch` `Stash` `Pop` | `Undo` | `Refresh`
   - 見た目: 小さなラベルボタン(既存モーダルのボタン様式)。区切りは縦線
   - **disabled 判定と理由**(ADR-0009 §4。hover で理由が分かるよう、disabled 時はラベル横に理由を出すスペースはないので
     `(disabled)` 風の薄色 + クリック時に footer へ理由を表示する方式):
     - Pull / Push: upstream なし → disabled(`no upstream`)。detached/unborn → disabled。
       Push は ahead=0 でも disabled(`nothing to push`)。Pull は behind=0 でも有効(fetch を兼ねるため)
     - Branch: 常に有効(HEAD から作成。選択 commit があればそこから — 既存 `open_create_branch_modal` に配線)
     - Stash: clean → disabled。dirty → 既存 `open_stash_push_modal` に配線
     - Pop: stash 0 → disabled。あれば**最新 stash の apply モーダル**を流用して開く(pop 化は T-HT-007)
     - Undo: ahead=0(= 全部 push 済み)or detached/unborn → disabled。**有効時もまだ未実装なので
       クリックで footer に「Undo: T-HT-009 で実装予定」を出すだけ**
     - Refresh: 常に有効 → `reload()` + footer 表示
   - **Pull / Push の実行も本チケットでは未配線**(クリックで footer に「T-HT-003/004 で実装」)
3. 右側表示(ADR-0010): `main ↑2 ↓0` / `no upstream` / `detached HEAD` / `no commits yet`。
   upstream 名はツールチップ的に省略可(StatusBar にある)。クリックで何もしない(later)
4. disabled 判定に使う値は `status_summary`(T-BP-003)を再利用
5. headless: `[kagi] toolbar: pull=<on|off> push=<on|off> stash=<on|off> pop=<on|off> undo=<on|off>` を reload 時にログ

## 実装メモ

### 変更ファイル
- `src/ui/mod.rs` — 唯一の変更ファイル

### Toolbar レイアウト
高さ 34px の flex row:
```
[RepoName] | [Pull] [Push] | [Branch] [Stash] [Pop] | [Undo] | [Refresh]  →  [branch ↑A ↓B]
```
右端の ahead/behind 表示は 4 形態: `main ↑2 ↓0` / `no upstream` / `detached HEAD` / `no commits yet`

### disabled 判定の実装
`StatusBarSummary` に `is_detached: bool`, `is_unborn: bool`, `stash_count: usize`, `repo_name: String` を追加。
`Head` enum のパターンマッチで `is_detached`/`is_unborn` を設定(文字列比較なし)。

`ToolbarState::toolbar_state()` で判定:
- `pull_on`: `!no_upstream` (no_upstream = detached/unborn/no upstream configured)
- `push_on`: `!no_upstream && ahead > 0`
- `stash_on`: `is_dirty`
- `pop_on`: `stash_count > 0`
- `undo_on`: `!not_attached && ahead > 0`

### headless ログ
`from_snapshot()` → `status_summary.toolbar_state().log_headless()` で毎回出力。
`reload()` は `from_snapshot` を呼ぶので自動的に更新・出力される。

### 配線状況
- Branch → `open_create_branch_modal(selected commit or HEAD)`
- Stash → `open_stash_push_modal`
- Pop → `open_stash_apply_modal(0)` (index 0 = 最新)
- Pull / Push / Undo: footer 案内のみ(後続チケット T-HT-003/004/009)
- Refresh: `reload()` + footer "Refreshed"

### 既存 Stash ボタンの扱い
旧 `render_header_slot` の条件付き Stash ボタンを削除し、Toolbar の Stash ボタンに一本化。挙動の取りこぼしなし。

### テスト結果
```
# fixture (dirty, main↑1, stash=1)
[kagi] toolbar: pull=on push=on stash=on pop=on undo=on

# clean repo (main↑0, no dirty)
[kagi] toolbar: pull=on push=off stash=off pop=off undo=off

# detached HEAD
[kagi] toolbar: pull=off push=off stash=off pop=off undo=off
```
