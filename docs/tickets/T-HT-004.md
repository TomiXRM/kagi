# T-HT-004: Push の plan + 実行(set-upstream flow 込み)

- Status: in-progress
- 依存: T-HT-003(ADR-0009)

## 背景

Toolbar の Push。**force / force-with-lease は実装しない**(ADR-0009)。
「実行前に push される commit 一覧を見せる」が要件の核。upstream 未設定時は set-upstream flow を提案する。

## スコープ

### 1. OperationPlan 拡張

- `pub preview_commits: Vec<String>` を追加(既存 plan_* では空。表示形式は `"<short>  <summary>"`)
- 共有モーダル `render_plan_modal_card` に「Commits to push (N)」セクションを追加
  (preview_commits 非空時のみ。最大10件 + `… and N more`。各行 truncate)

### 2. plan_push / execute_push(ops.rs 追記)

- `pub fn plan_push(repo: &Repository) -> Result<OperationPlan, GitError>`
  - blocker: detached / unborn / **ahead 0 かつ upstream あり**(nothing to push)/
    upstream なしかつ remote も無い
  - **set-upstream flow**: upstream 未設定で remote(origin 優先、なければ唯一の remote)がある場合は
    blocker にせず、title を `Push '<branch>' to '<remote>' (set upstream)` にして実行可能にする
  - preview_commits: `upstream..HEAD`(set-upstream 時は HEAD から到達可能な全 commit、上限 100)を
    revwalk で列挙(新しい順)
  - warning: 「non-fast-forward の場合は失敗します(force はしません)」
  - dirty WT は push に無関係なので blocker にしない
- `pub fn execute_push(repo: &Repository, repo_path: &Path) -> Result<PushOutcome, GitError>`
  - upstream あり: `run_git(repo_path, ["push", "<remote>", "<branch>"])`
  - upstream なし: `run_git(repo_path, ["push", "-u", "<remote>", "<branch>"])`
  - 失敗(non-FF 含む): stderr を含めて GitError(**自動 force 昇格しない**)。ローカルは何も変わらない
  - `PushOutcome { pushed: usize, set_upstream: bool }`(pushed は plan 時の件数でよい)
- preflight: 既存 preflight_check

### 3. UI / env

- Push ボタン → plan モーダル(preview commits 付き)→ confirm → execute → oplog → reload + footer
- toolbar の push_on 判定を更新: `(upstream あり && ahead>0) || (upstream なし && attached && remote あり)`
- `KAGI_PUSH=1`(+ KAGI_AUTO_CONFIRM=1)

### 4. テスト(tests/push_test.rs 新規。pull_test.rs の bare remote ヘルパー構成を踏襲)

- ahead 2 → push 成功、remote の ref が HEAD に一致、preview_commits が2件
- ahead 0 → blocker
- upstream なし + origin あり → set-upstream plan → execute 後 upstream が設定され ahead/behind が出る
- non-FF(remote が先に進んでいる)→ Failed + stderr に rejected、ローカル無傷
- detached → blocker
- **force が絶対に走らない**: execute_push の引数組み立てに "--force" 系が含まれないことを grep でも確認

| 条件 | 結果 |
|------|------|
