### 2. Stash Pop(ADR-0009 の Destructive 緩和付き)

- `pub fn plan_stash_pop(repo: &mut Repository, index: usize) -> Result<OperationPlan, GitError>`
  - blocker: index 範囲外 / conflict 状態 / **dirty WT**(apply と同じポリシー)/
    **conflict 予測あり**(stash commit と HEAD の in-memory merge(`merge_commits` 等)で判定。
    conflict 予測時は「pop すると stash が消えるため拒否。apply を使ってください」と明記)
  - 表示: 「pop = apply + drop(成功時に stash が消える)」の説明 / 対象 stash の message
- `pub fn execute_stash_pop(repo: &mut Repository, index: usize) -> Result<(), GitError>`
  - `repo.stash_apply(index, None)` → 成功時のみ `repo.stash_drop(index)`
  - apply が失敗したら drop しない(stash 無傷)
  - ※ stash_drop は ADR-0004 で Destructive だが、ADR-0009 で「pop の一部としてのみ」許可された経緯を
    doc comment に明記。**単独の stash_drop API は公開しない**(pub にしない)
- テスト(`tests/stash_pop_test.rs` 新規):
  - pop 正常系: 変更が戻り stash が消える(count-1)
  - conflict 予測 → blocker + stash 残存 + repo 無傷
  - dirty → blocker / 範囲外 → blocker
  - apply 失敗時に drop されない(構成が難しければ conflict 予測 blocker で代替し、その旨メモ)
  - push の include_untracked=false で untracked が残る
  - 計 6+ ケース、tempdir のみ

## 実装メモ (T-HT-007 完了時点)

### シグネチャ変更と既存テストの最小修正

`plan_stash_push` / `execute_stash_push` に `include_untracked: bool` 引数を追加。
既存の呼び出し箇所 (src/ui/mod.rs, src/main.rs, tests/ops_test.rs) に `true` を追加する最小修正を実施。
テストのロジックは一切変更していない。変更理由: チケット「シグネチャ追従の最小修正のみ可」に準拠。

### conflict 予測の実装方法

`predict_stash_pop_conflict(repo, head, stash_oid)` を追加。

- `stash_oid` は `stash_foreach` コールバックで受け取る Oid (stash commit そのもの)
- `repo.merge_commits(&head_commit, &stash_commit, None)` で in-memory merge
- stash commit の親構造: parent[0] = stash 前の HEAD (base), parent[1] = index snapshot,
  parent[2] = untracked files commit (INCLUDE_UNTRACKED 時)
- merge_commits は stash commit (working-tree 状態) と現在 HEAD を対象にするため、
  apply 後の状態を正確に予測できる
- conflict あり → blocker メッセージ + 「Stash Apply を使ってください」提案
- in-memory のみ: WT / repo state は一切変更しない

`plan_stash_pop` 内で `collect_stash_entries_with_oid` を呼び、OID を先に取得してから
`predict_stash_pop_conflict` に渡すことで `&Repository` の制約問題を回避した
(stash_foreach は `&mut Repository` を要求するため)。

### execute_stash_pop の設計

```
repo.stash_apply(index, None)  // apply
  → 失敗: Err返却、stash_drop 呼ばず(stash無傷)
  → 成功: stash_drop_internal(repo, index)
```

`stash_drop_internal` は `fn`(private) で `pub fn` にしていない。
ADR-0004 (Destructive 操作ポリシー) / ADR-0009 の経緯を doc comment に明記。
