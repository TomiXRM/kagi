# T-HT-009: Undo Commit backend(ref 付け替えのみの soft 相当)

- Status: in-progress(worktree 並行)
- 依存: ADR-0011
- 範囲: **backend + テストのみ**(UI 配線は PM が main 側で実施)

## スコープ(ADR-0011 を厳守)

- `pub fn plan_undo_commit(repo: &Repository) -> Result<OperationPlan, GitError>`
  - blocker: detached / unborn / conflict 状態 / HEAD が merge commit(parents>1)/
    **HEAD が upstream から到達可能(= push 済み)**(upstream なしなら無条件で undo 可)/
    HEAD が root commit(親なし)
  - 表示: 取り消される commit(short + summary)を title or warnings に。
    「変更は staged のまま残ります(何も失われません)」を明記
  - recovery: `git reset --soft <元sha>` で完全復元可能な旨 + 元 sha
- `pub fn execute_undo_commit(repo: &Repository) -> Result<UndoOutcome, GitError>`
  - **ref 付け替えのみ**: `repo.reference("refs/heads/<branch>", parent_oid, true, msg)`。
    **index・working tree・HEAD(symbolic)には一切触れない**(checkout 系 API 禁止)
  - `UndoOutcome { undone: CommitId(元sha), now_at: CommitId(親sha) }`
- テスト(`tests/undo_test.rs` 新規):
  - 正常系: HEAD~1 に戻り、**取り消した commit の変更が staged で残る**(`git status --porcelain` で A/M が INDEX 側)、WT ファイル内容は不変
  - push 済み(upstream = HEAD)→ blocker / merge commit → blocker / root commit → blocker / detached → blocker
  - upstream なしローカル branch → undo 可
  - undo → `git reset --soft <元sha>`(CLI)で完全復元できる(round-trip)
  - 計 7+ ケース、tempdir のみ

## 作法

- ops.rs の既存パターン(plan_pull 等)に完全に合わせる。preflight は既存 preflight_check を再利用
- OperationPlan の preview_commits フィールドが merge 時に入る可能性がある(T-HT-004 と並行のため)。
  **この worktree の時点の OperationPlan 定義に従い、構築箇所はフィールド名を明示した struct リテラルにしない**
  (既存 plan_* と同様にフィールド逐次代入 or 既存形式を踏襲 — merge 競合を最小化)

## 実装メモ (2026-06-12 完了)

### 実装した関数

- `UndoOutcome { undone: CommitId, now_at: CommitId }` — ops.rs に追加
- `plan_undo_commit(repo) -> Result<OperationPlan, GitError>` — blocker 判定 + plan 生成
- `execute_undo_commit(repo) -> Result<UndoOutcome, GitError>` — ref 付け替えのみ

### blocker 判定の実装

push 済み判定には `graph_descendant_of(upstream_oid, head_oid)` を使用。
これは「upstream が head の子孫である → head は upstream から到達可能 → push 済み」を意味する。
加えて `upstream_oid == head_oid`(同一コミット)の場合も blocker。
upstream が未設定(local-only ブランチ)の場合はこのチェックをスキップする。

### ref 付け替えの実装

```rust
repo.reference(&branch_refname, parent_oid, true, &log_msg)
```

`branch_refname` は `"refs/heads/<branch>"` で、HEAD の symbolic ref はそのまま維持。
`checkout_tree` / `set_head` / `reset_default` / `index.*` は一切呼んでいない。
undone commit の変更は git2 のインデックス操作なしに自然に staged 状態で残る。
