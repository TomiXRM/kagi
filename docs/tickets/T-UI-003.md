# T-UI-003: diff を main pane に全幅表示(ユーザー要望)

- Status: in-progress
- 依存: T012, T025

## 背景

ユーザー提案: 「コードはサイドバー幅で読めない。ファイルをクリックしたら commit graph が出ている main pane に diff を表示したい」。
現状の「右パネル内 diff(560px)」と「Commit Panel 中段の diff ボックス」を廃止し、main pane 切替方式へ。

## スコープ

1. 状態: `KagiApp.main_diff: Option<MainDiffView>`
   ```rust
   pub struct MainDiffView {
       pub title: SharedString,        // ファイル名
       pub stats: SharedString,        // "+12 −3"
       pub rows: Vec<DiffRow>,         // 既存 FileDiffView::rows を流用
       pub source: MainDiffSource,     // 再読込・戻り先判定用
   }
   pub enum MainDiffSource {
       Commit { row_index: usize, file_index: usize },
       Unstaged { path: PathBuf },
       Staged { path: PathBuf },
   }
   ```
2. ルーティング変更:
   - 詳細パネルの changed file クリック → `open_main_diff(Commit{..})`(従来の file_diff_view への切替は廃止。
     `file_diff_view` フィールドと diff パネル描画は削除 or 未使用化 — 削除する場合は KAGI_OPEN_FIRST_FILE の
     headless ログ互換を維持: 同じ `[kagi] diff: <path> hunks=N (+A -R)` を main diff 経由で出す)
   - Commit Panel のファイルクリック → `open_main_diff(Unstaged/Staged{..})`。
     Commit Panel 中段の diff ボックスは削除し、その分 Unstaged/Staged 箱を大きく(1:1 は維持)
3. main pane 描画: `main_diff` が Some のとき、commit list 列(カラムヘッダ + WIP行 + uniform_list)を
   **全幅 diff view** に置き換え:
   - ヘッダ行: `← Back`(クリックで `main_diff = None`)+ ファイル名 + stats(+N −M)
   - 本体: `uniform_list`(id "main-diff-list")。行 = hunk ヘッダ(青)or コード行:
     `old_lineno | new_lineno | sigil+content`(行番号は各4〜5桁の固定幅・muted、Added は new のみ、
     Removed は old のみ。DiffLine に lineno は既にある)
   - 色: 既存の diff 配色(緑/赤背景)
4. **Esc で閉じる**: `actions!` に CloseMainDiff を追加し `escape` を bind(main_diff が None のときは何もしない)。
   sidebar / 右パネル / Status Bar / Bottom Panel は diff 表示中もそのまま
5. 選択中 commit が変わったら main_diff を閉じる。reload() でも閉じる(stale 防止)
6. headless: 既存 `KAGI_SELECT_FIRST=1 KAGI_OPEN_FIRST_FILE=1` が main diff 経由で従来同等のログを出す +
   `[kagi] main-diff: open <path> rows=N` を追加
