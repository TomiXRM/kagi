# T-UI-004: diff のシンタックスハイライト(tree-sitter、外部バイナリなし)

- Status: in-progress
- 依存: T-UI-003(背景: ユーザーと delta バイナリ依存の問題を議論 → gpui-component 同梱の highlighter を採用)

## スコープ

1. `Cargo.toml`: gpui-component の feature に `tree-sitter-languages` を追加(**このチケットのみ Cargo.toml 変更可。
   依存クレートの追加はなし、feature 追加のみ**)。ビルド時間増に注意(初回 +数分)
2. `gpui_component::highlighter` の API を registry ソース(`~/.cargo/registry/src/*/gpui-component-0.5.1/src/highlighter/`)で
   調査し、**main diff の DiffRow::Line のコード部分**をハイライトする:
   - 言語判定: ファイル拡張子(rs / py / ts / js / json / toml / md / sh / c / cpp / css / html 等。
     未対応拡張子はハイライトなしで従来表示)
   - 各行の content を per-line でハイライトして styled runs(`StyledText` / `TextRun` 等 gpui の方法)で描画。
     行単位ハイライトで文脈が崩れるのは許容(diff 表示の通例)。
     もし highlighter API がファイル全体前提なら、diff の new 側テキストを結合して一括ハイライト → 行ごとに分配する方式でもよい(実装しやすい方。メモに明記)
   - +/− の背景色(緑/赤)と行番号は現状維持。前景色だけハイライトの色に
   - テーマ: gpui-component の HighlightTheme デフォルト(dark)で開始し、可能なら背景透過で
     うちの配色に馴染ませる
3. ハイライト計算は **main diff を開いた時に1回**(MainDiffView 構築時)。描画ループで毎回やらない
4. 失敗時(unknown 言語 / highlighter エラー)は無色フォールバック(クラッシュ禁止)
5. headless: `[kagi] main-diff: open <path> rows=N highlight=<lang|none>` にログ拡張

## 実装メモ

### 採用 API

`gpui_component::highlighter::SyntaxHighlighter` (tree-sitter ベース) を採用。
`ropey::Rope` は `gpui_component::Rope` として re-export されているため、`Cargo.toml` への新規依存追加なしで利用可能。

### ハイライト方式

**一括ハイライト → 行ごとに分配** 方式を採用。

1. `open_main_diff_commit` / `open_main_diff_wip` の MainDiffView 構築直後に
   `highlight_diff_rows(&mut rows, &path)` を1回呼ぶ。
2. `highlight_diff_rows` 内では全 Line 行のコンテンツ(先頭シジル +/-/ を除く)を
   `\n` 区切りで結合した文字列を `Rope` に変換し、`SyntaxHighlighter::update` → `styles` を1回実行。
3. 得られた `(byte_range, HighlightStyle)` を各行のバイトオフセットを追跡しながら
   `DiffRow::Line.highlights` フィールドに分配(行ローカルオフセット、シジル1バイト分ずらし)。
4. 描画時は `StyledText::new(text).with_highlights(highlights)` を使用。
   `highlights` が空の場合は従来の plain text フォールバック。

### Cargo.toml 変更

```
gpui-component = { version = "0.5.1", features = ["tree-sitter-languages"] }
```

### 言語判定

ファイル拡張子 → `lang_for_ext()` で言語名文字列を決定。
rs / py / js / jsx / ts / tsx / json / toml / yaml / yml / md / sh / c / cpp / h / hpp / css / html / go / java / rb / zig / sql / swift に対応。
未対応拡張子は `highlight=none` でフォールバック(クラッシュなし)。

### テーマ

`HighlightTheme::default_dark()` を使用(Catppuccin Mocha 配色に近い暗色テーマ)。
前景色のみ置き換え(background_color は `None` のまま保持)。
+/- 背景色(BG_DIFF_ADDED / BG_DIFF_REMOVED)および行番号列は変更なし。
