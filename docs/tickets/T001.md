## 背景

プロジェクトの土台として、GPUI ウィンドウが開くだけの最小アプリを作る。以降の全 UI ticket の前提。
本 ticket では Git 機能・UI コンポーネントの作り込みは一切行わない。

## 事前調査で確定済みの技術情報(PM 調査 2026-06-12)

- gpui は crates.io 公開済み、最新 **0.2.2**。`Cargo.toml` は次で固定する:
  ```toml
  gpui = { version = "0.2.2", features = ["runtime_shaders"] }
  ```
- **`runtime_shaders` feature は必須**。本開発機は Xcode Command Line Tools のみで `xcrun metal` が使えないため、デフォルトのビルド時 Metal シェーダコンパイルは失敗する。
- 0.2.2 の最小コードの形(docs.rs 同梱 hello_world.rs より):
  - `Application::new().run(|cx: &mut App| { ... })`
  - `cx.open_window(WindowOptions { window_bounds: Some(WindowBounds::Windowed(bounds)), ..Default::default() }, |_, cx| cx.new(|_| MyView { ... }))`
  - view は `impl Render for MyView`、`render(&mut self, _window: &mut Window, _cx: &mut Context<Self>) -> impl IntoElement` で `div()...` を返す
  - `Bounds::centered(None, size(px(800.), px(600.)), cx)` でウィンドウ位置指定
  - 最後に `cx.activate(true)`
- API の詳細は https://docs.rs/gpui/0.2.2 と docs.rs のソース内 examples を参照(git main の examples は API が違うので見ない)。

## 実装メモ

### 使った API (gpui 0.2.2)

- `Application::new().run(|cx: &mut App| { ... })` でアプリ起動
- `Bounds::centered(None, size(px(800.), px(600.)), cx)` でウィンドウ中央配置
- `cx.open_window(WindowOptions { window_bounds: Some(WindowBounds::Windowed(bounds)), ..Default::default() }, |_, cx| cx.new(|_| KagiApp { .. })).unwrap()` でウィンドウ作成
- `cx.activate(true)` でアプリをフォアグラウンドに
- `impl Render for KagiApp` + `fn render(&mut self, _window: &mut Window, _cx: &mut Context<Self>) -> impl IntoElement` でビュー実装
- `div().flex().flex_col().items_center().justify_center().size_full().bg(...).child(...)` でレイアウト
- 主要 import: `use gpui::{App, Application, Bounds, Context, SharedString, Window, WindowBounds, WindowOptions, div, prelude::*, px, rgb, size};`

### Cargo.toml

```toml
gpui = { version = "0.2.2", features = ["runtime_shaders"] }
```

- `runtime_shaders` feature は必須。これがないと `build.rs` が `xcrun metal` を呼び出して失敗する（Xcode Command Line Tools のみの環境ではコンパイル不可）。

### 確認した事項

- `cargo check`: 成功 (39秒)
- `cargo build`: 成功 (22秒、依存クレートのビルド済みキャッシュあり)
- `cargo run`: バックグラウンドで起動し、10秒間プロセスが生存することを確認 (PID が `target/debug/kagi` として存在)。クラッシュなし。

### ハマった点

- 特になし。PM 事前調査の API 情報が正確だったため、そのまま実装できた。
- git main ブランチの examples は `gpui_platform` 分割後の API が混入しているため参照しないよう注意。docs.rs/gpui/0.2.2 が正。
- macOS では `block v0.1.6` に future-incompat warning が出るが、ビルド・実行には影響なし。
