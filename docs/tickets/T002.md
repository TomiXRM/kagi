# T002: repo path を指定して git repository を開く

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T001

## 背景

Git Backend Layer(ADR-0002: git2 採用)の第一歩。リポジトリを開けることは以降の全 ticket の前提。
このチケットでは「開いて基本情報を表示する」だけに絞り、status / log の取得は T003 / T004 で行う。

## スコープ

1. `git2` crate を導入する。**MVP はローカル操作のみで https/ssh トランスポート不要**のため、可能なら `default-features = false` で openssl 等のビルドを回避する(最新バージョンと feature 構成は docs.rs / crates.io で確認すること)。
2. `src/git/mod.rs` に repo open 機能を実装する:
   - `pub fn open_repository(path: &Path) -> Result<RepoInfo, GitError>`
   - 検証: パスが存在しない / git repo でない / bare repo → それぞれ別個のわかりやすいエラー
   - `RepoInfo` には最低限: repo 名(workdir のディレクトリ名)、workdir パス、HEAD 状態
   - HEAD 状態は architecture.md §3 の `Head` enum(Attached / Detached / Unborn)をここで導入する
3. UI: 起動時コマンドライン引数で path を受け取る(`cargo run -- /path/to/repo`)。
   - 開けたら: repo 名・パス・HEAD(branch 名 or detached SHA or unborn)をウィンドウに表示
   - 引数なし: 「Usage: kagi <repo-path>」的な案内を表示(クラッシュしない)
   - エラー時: エラーメッセージをウィンドウに表示(クラッシュしない)
4. open は起動時に1回だけなので**同期実行でよい**。バックグラウンドワーカー化は T003 以降で行う。

## テスト方法

1. `bash scripts/make_fixture.sh` で fixture を生成(最終行が repo パス)
2. `cargo run -- <fixtureのrepoパス>` → repo 名 "repo" と HEAD "main" が表示される(目視 or プロセス生存 + 表示文字列のログ出力で代替可)
3. `cargo run -- /tmp/does-not-exist` / `cargo run -- /tmp`(非repo)/ `cargo run -- <fixture>/remote.git`(bare)→ 各エラー表示
4. `mktemp -d` した空ディレクトリで `git init` した unborn repo を開く → Unborn 表示
5. **検証はすべて fixture / tempdir 配下の repo で行うこと。本プロジェクト repo や他の既存 repo への書き込みは禁止(読み取りで開くのも fixture を使う)**

## リスク

- libgit2 の C ビルドが走るため初回ビルドが伸びる(timeout 600000ms 推奨)
- git2 の feature 構成を誤ると openssl のビルドでつまずく可能性 → その場合は default features に戻して報告
- HEAD unborn の扱いは git2 では `Repository::head()` がエラーを返すため、エラー種別の判定が必要

## 実装メモ

### 実装日時
2026-06-12

### git2 バージョンと feature 構成
- `git2 = { version = "0.21", default-features = false }`
- git2 0.21.0 の default feature は空集合なので `default-features = false` で問題なし
- openssl / ssh / https のビルドは発生しない
- 実際に追加された依存: `git2 v0.21.0`, `libgit2-sys v0.18.5+1.9.4`, `libz-sys v1.1.29`, `vcpkg v0.2.15`

### 変更ファイル一覧
- `Cargo.toml` — git2 依存を追加
- `Cargo.lock` — 自動更新
- `src/git/mod.rs` — 新規作成: `open_repository`, `RepoInfo`, `Head`, `GitError`
- `src/main.rs` — CLI 引数処理 + repo 情報表示に更新

### API の要点
- `open_repository(path)` → `Result<RepoInfo, GitError>`
- `GitError` の区別: `PathNotFound` / `NotARepository` / `BareRepository` / `Other`
- HEAD 判定:
  - `repo.head()` が `Ok` かつ `is_branch()` → `Head::Attached`
  - `repo.head()` が `Ok` かつ非 branch → `Head::Detached`
  - `repo.head()` が `Err(UnbornBranch)` → `HEAD` reference の `symbolic_target()` から branch 名を取得 → `Head::Unborn`
- `symbolic_target()` は git2 0.21 では `Result<Option<&str>, Error>` を返すため `.ok().flatten()` でネストを展開
