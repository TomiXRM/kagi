## スコープ

1. ドメインモデルを `src/git/` に追加(例: `src/git/refs.rs`):
   - `Branch { name: String, target: CommitId, upstream: Option<UpstreamInfo> }`
   - `UpstreamInfo { remote_branch: String, ahead: usize, behind: usize }`(`graph_ahead_behind` で算出)
   - `RemoteBranch { remote: String, name: String, target: CommitId }`("origin/main" → remote="origin", name="main")
   - `Tag { name: String, target: CommitId }`(annotated tag は commit まで peel。commit 以外を指す tag は skip)
   - `Stash { index: usize, message: String, target: CommitId }`(読み取りのみ。stash 操作は T015)
2. snapshot 関数(例: `src/git/snapshot.rs`):
   - `pub struct RepoSnapshot { head: Head, commits: Vec<Commit>, branches: Vec<Branch>, remote_branches: Vec<RemoteBranch>, tags: Vec<Tag>, status: WorkingTreeStatus, stashes: Vec<Stash> }`
   - `pub fn snapshot(repo: &mut Repository, commit_limit: usize) -> Result<RepoSnapshot, GitError>`
     (`&mut` は `stash_foreach` が要求する場合のみ。不要なら `&`)
   - 既存の `resolve_head` / `commit_log` / `working_tree_status` を内部で再利用する(資材の重複実装をしない。`resolve_head` の可視性調整は可)
   - branch 列挙: `repo.branches(Some(BranchType::Local))` / `Remote`。remote の HEAD エイリアス(`origin/HEAD`)は除外
   - upstream: `branch.upstream()` が Ok のときのみ `graph_ahead_behind(local, upstream)` で ahead/behind
   - stash: `repo.stash_foreach(...)`
3. `src/main.rs` をリファクタ: open 後に **snapshot を1回だけ**取得して全表示を snapshot 経由にする(Repository の二重 open 解消)
4. UI 表示(最小限・テキストで可):
   - `BRANCHES:` 各行 `main ↑1 ↓0 [HEAD]` 形式(ahead/behind は upstream ありのみ、HEAD マーク付き)
   - `REMOTE BRANCHES:` / `TAGS:` / `STASHES:` の一覧
5. integration test(`tests/snapshot_test.rs`):
   - tempdir + git CLI で bare remote 付き repo を組み立て(scripts/make_fixture.sh と同様の構成。スクリプト自体は変更禁止だが参考に読んでよい):
     - main が ahead 1 / behind 0、別 branch が behind 1 になる構成
     - lightweight tag と annotated tag を1つずつ
     - stash 1件
   - assert: branches(upstream の ahead/behind 値含む)/ remote_branches(origin/HEAD 除外含む)/ tags(annotated が commit に peel されること)/ stashes(index, message)/ head が Attached("main")

## リスク

- `graph_ahead_behind` の引数順(local, upstream)を逆にすると ahead/behind が逆転する → fixture の ↑1/↓1 で検出可能
- `stash_foreach` は `&mut Repository` を要求する(API シグネチャに注意)
- annotated tag の peel は `reference.peel_to_commit()` が簡単。commit 以外(tree/blob への tag)は skip
- `origin/HEAD` は symbolic ref。`Branch::name()` が "origin/HEAD" を返す場合は除外する

### 設計判断

1. **upstream 名の扱い**: `branch_ref.upstream().name()` は `"origin/main"` のような short name を返す。upstream が symbolic ref (target = None) の場合は ahead/behind 計算をスキップして `None` を返す。

2. **origin/HEAD 除外**: `collect_remote_branches` で `full_name.ends_with("/HEAD")` でフィルタ。加えて symbolic ref (OID target が None) も除外しているため二重に安全。

3. **annotated tag の peel**: `repo.tag_foreach` は各 tag の raw OID を渡す。`repo.find_object(oid, None).peel_to_commit()` で annotated tag も lightweight tag も統一的に commit まで peel できる。commit まで peel できない tag は skip。

4. **stash_foreach**: `&mut Repository` を要求するため `snapshot()` のシグネチャは `&mut Repository`。

5. **upstream 名**: `branch_ref.upstream().name()` は "origin/main" 形式の short name (refs/remotes/ prefix なし) を返す。architecture.md §3 の `UpstreamInfo.remote_branch` 仕様と一致。
