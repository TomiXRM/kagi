# T007: graph layout の網羅テストを作る

- Status: in-progress

## 背景

graph layout は本プロダクトの中心であり、T009 の描画はこの出力を盲信する。
T006 のサニティテストを超えて、トポロジーの網羅ケースと**不変条件チェッカー**で品質を固定する。
このチケットは**テストのみ**。layout 本体のバグを見つけた場合は修正せず報告する(PM が判断する)。

## スコープ

`tests/graph_layout_test.rs` を新規作成(public API `kagi::graph::layout` / `kagi::git::{Commit, CommitId, Signature}` のみ使用)。

### 1. 不変条件チェッカー(全ケースに適用するヘルパー)

```rust
fn check_invariants(commits: &[Commit], gl: &GraphLayout)
```

- rows.len() == commits.len()、rows[i].commit == commits[i].id(順序保存)
- 各行: IntoNode はすべて to_lane == row.lane、OutOfNode はすべて from_lane == row.lane、Pass は from == to かつ != row.lane
- **edge 連続性**: 各隣接行 (r, r+1) について、「r の下端で開いている lane 集合」(Pass∪OutOfNode の to_lane)と「r+1 の上端から来る lane 集合」(Pass∪IntoNode の from_lane)が一致する
- lane_count == 全行を通じた max(使用 lane index)+1 以上、かつ全 edge の lane が lane_count 未満
- 同一行内で同一 (from, to, kind) の edge が重複しない

### 2. トポロジーケース(各ケースで check_invariants + 形状の個別 assert)

| ケース | 検証ポイント |
|--------|--------------|
| 直線 10 commits | 全行 lane 0、Pass なし |
| 分岐して merge(fixture 同型) | T006 test 3 と同等 + 不変条件 |
| **octopus merge(parents 3つ)** | OutOfNode が3本、lane_count == 3、lane 爆発なし |
| **criss-cross merge**(X←A,B / Y←A,B 型) | 不変条件が保たれる(形状は assert しすぎない) |
| 並行する長い branch(共有 root まで 5 行併走) | 併走区間の全行に Pass edge がある |
| 複数 root + 併走(root が 2 つ、片方 active 中にもう片方が始まる) | 2 lane 使用後、解放された lane の**再利用**を確認 |
| merge 後の lane 解放→新 branch tip が lane 再利用 | find_or_push_free_lane の最左性 |
| 単一 commit(root のみ) | rows 1 件、edges 空、lane_count 1 |
| stress: 三層の分岐 merge ネスト(手書き 12〜15 commits 程度) | 不変条件のみ |

### 3. end-to-end ケース(git2 経由)

- tempdir + git CLI で分岐 + merge + 並行 branch の repo を組み立て(既存 tests/log_test.rs のヘルパー作法を踏襲)
- `open → commit_log → layout` を通し、check_invariants が成立することを確認
- merge commit の行に OutOfNode が2本あることを確認
