# T008: GPUI で commit list を表示する(仮想化リスト)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T001, T005

## 背景

これまでの UI は stderr ログ + テキスト羅列だった。本チケットで Graph Panel の土台となる
**仮想化された commit リスト**を作る(非機能要件: 10k commits でもスクロールが軽い)。
lane 描画は T009、行選択は T010 で重ねるため、ここでは「リストが出る」ことに集中する。

## スコープ

1. UI モジュール構成の導入:
   - `src/ui/mod.rs` + `src/ui/commit_list.rs`(バイナリ側 `main.rs` から `mod ui;` で参照。lib.rs に入れない: テスト対象は domain/backend のみ)
   - `KagiApp`(root view)が `RepoSnapshot` を保持し、commit list を子要素として描画
2. commit list(`uniform_list` を使用):
   - **gpui 0.2.2 の `uniform_list` API を docs.rs で必ず確認してから実装**(シグネチャを想像で書かない)
   - 各行: `short id`(等幅・薄色)/ summary(主テキスト、1行 truncate)/ author 名 / 日付
   - 日付は epoch 秒から自前整形でよい(「3d ago」「2y ago」程度の相対表記。**新規依存 crate は追加しない**。SystemTime::now との差分で計算)
   - 行高は固定(uniform_list の前提)
3. ref バッジ:
   - snapshot から `CommitId → Vec<RefBadge>` map を作るヘルパーを `src/ui/` に実装
   - badge: HEAD(強調色)/ local branch / remote branch / tag を色違いの小さなチップで commit 行に表示
   - HEAD が attached の場合、その branch チップに HEAD マークを統合してよい(`main ✓` 等)
4. レイアウト: ウィンドウ全体を縦分割 — 上部に repo 名 + HEAD + status 概要のヘッダ(1行)、残り全部が commit list
5. エラー/引数なし表示は従来どおり(クラッシュしない)
6. stderr ログ([kagi] 行)は検証手段として維持する(リスト行数のログ等)

### 使った gpui API

#### `uniform_list` シグネチャ (gpui 0.2.2)

```rust
pub fn uniform_list<R>(
    id: impl Into<ElementId>,
    item_count: usize,
    f: impl 'static + Fn(Range<usize>, &mut Window, &mut App) -> Vec<R>,
) -> UniformList
where
    R: IntoElement,
```

ソース: `~/.cargo/registry/src/index.crates.io-1949cf8c6b5b557f/gpui-0.2.2/src/elements/uniform_list.rs`  
公式例: `~/.cargo/registry/src/index.crates.io-1949cf8c6b5b557f/gpui-0.2.2/examples/uniform_list.rs`

#### 実際の呼び出しパターン

```rust
uniform_list(
    "commit-list",
    row_count,
    cx.processor(move |this, range, _window, _cx| {
        render_rows(&this.rows, range)
    }),
)
.flex_1()
.h_full()
.w_full()
```

`cx.processor` は `Context<T>` のメソッドで、シグネチャ:
```rust
pub fn processor<E, R>(
    &self,
    f: impl Fn(&mut T, E, &mut Window, &mut Context<T>) -> R + 'static,
) -> impl Fn(E, &mut Window, &mut App) -> R + 'static
```
これにより `uniform_list` の第3引数 (クロージャ型 `Fn(Range<usize>, &mut Window, &mut App) -> Vec<R>`) に変換できる。

#### 使用した Styled / layout メソッド
- `div().flex().flex_col().flex_row().flex_1().flex_shrink_0()` — flex レイアウト
- `.w(px(...)).h_full().size_full()` — サイズ指定
- `.px_3().py(px(3.)).px_1().py_1().mr_2().gap_1()` — spacing
- `.bg(rgb(...)).text_color(rgb(...))` — 色指定
- `.text_sm().text_xl().overflow_hidden()` — テキスト
- `.rounded_sm()` — badge chip の角丸
- `.into_any()` — `impl IntoElement` → `AnyElement` 変換 (Render の型統一)

### 注意点

1. **`into_any()`が必要**: `Render::render` が `impl IntoElement` を返す型を if/else で分岐させる場合、両アームの型を統一するために `.into_any()` で `AnyElement` に変換する
2. **SharedString はスナップショット取得時に生成**: `build_commit_rows` を一度だけ呼び、`CommitRow` は全フィールドを `SharedString` で保持。render クロージャ内では `.clone()` のみ
3. **`cx.processor` が唯一の正しい方法**: `uniform_list` の第3引数は `Fn(Range, &mut Window, &mut App)` で view state への参照を持てない。`cx.processor` が view を内部で `Entity::update` 経由でアクセスするラッパを返す
4. **uniform_list の行高**: `measure_item` が最初の可視アイテムをレイアウトして高さを決定するため、行内レイアウトは固定高に収める必要がある。`py(px(3.))` 固定で対応
5. **日付計算**: `SystemTime::now()` から Unix epoch 秒を取得し自前で相対表記に変換。新規 crate 追加なし
