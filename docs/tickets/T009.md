# T009: GPUI で commit graph lane を描画する

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T006, T007, T008

## 背景

T006 の `GraphLayout` を T008 の commit list に接続し、各行の左側に lane(node ● と edge)を
描画する。これで「コミットグラフ中心」という本プロダクトの中心 UI が初めて姿を現す。
描画は **T006 の出力を盲信してよい**(品質は T007 で固定済み)。

## スコープ

1. layout の統合:
   - snapshot 取り込み時(`KagiApp::from_snapshot`)に `kagi::graph::layout(&snap.commits)` を1回実行
   - `CommitRow` に行の `lane: usize` と `edges: Vec<GraphEdge>` を保持(描画用に必要なら事前変換した形でよい)
   - `[kagi] graph: lane_count=N` を stderr ログに出す
2. 行内の lane 描画:
   - 各 commit 行の先頭(SHA の左)に固定幅の graph 領域を設ける
     - 幅 = `min(lane_count, 8) * LANE_W`(LANE_W ≈ 14px)。8 lane 超は描画を clip してよい(MVP)
   - **gpui 0.2.2 の描画 API(`canvas` 要素 / `PathBuilder` / `window.paint_path` 等)を docs.rs と
     `~/.cargo/registry/.../gpui-0.2.2/` のソース・examples(例: `examples/painting.rs` があれば)で必ず確認してから実装**
   - 描画内容(行ローカル座標系。行の上端 y=0、下端 y=row_h):
     - node: `row.lane` の x 中心、y 中央に小さい塗り circle(半径 ≈ 4px)
     - edge: kind ごとに
       - Pass: lane x で上端→下端の縦線
       - IntoNode: 上端の from_lane x → node 中心への線(from==lane なら縦線、異なれば斜め/折れ線)
       - OutOfNode: node 中心 → 下端の to_lane x への線
     - 線は直線で十分(ベジェ曲線は任意。やりすぎない)
   - 色: lane index ベースのパレット(6〜8色を循環)。node は自 lane 色、edge は
     IntoNode → from_lane の色 / OutOfNode → to_lane の色 / Pass → 自 lane の色
3. 行の総高さは固定を維持(uniform_list 前提)。行高は現状値を測って定数化してよい
4. unborn / エラーケースは従来どおり(graph 領域なしで動く)

### 使った描画 API

```rust
// ストローク線(edge)
let mut builder = PathBuilder::stroke(px(EDGE_W));
builder.move_to(point(px(x0), px(y0)));
builder.line_to(point(px(x1), px(y1)));
if let Ok(path) = builder.build() {
    window.paint_path(path, color);  // color: Hsla (Into<Background>)
}

// 塗り潰し円(node)
let mut builder = PathBuilder::fill();
// move_to / line_to で 12 点多角形近似 → close()
if let Ok(path) = builder.build() {
    window.paint_path(path, color);
}

// canvas 要素のシグネチャ
canvas(
    |bounds: Bounds<Pixels>, window: &mut Window, cx: &mut App| -> T { ... }, // prepaint
    |bounds: Bounds<Pixels>, prepaint: T, window: &mut Window, cx: &mut App| { ... }, // paint
) -> Canvas<T>
```

`Bounds<Pixels>` の絶対座標は `f32::from(bounds.origin.x)` で取得(`.0` フィールドは `pub(crate)`).

`Canvas<T>` は `Styled` を実装しているので `.size_full()` などのスタイルメソッドが呼べる.

### 色パレット

6色の HSL パレット(h: 0.583/0.333/0.083/0.917/0.750/0.500, s=0.75, l=0.65, a=1.0)を `lane % 6` で循環.
