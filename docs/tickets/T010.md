# T010: commit selection で metadata panel を表示する

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T008, T009

## 背景

requirements M-05 の前半。「commit を選択すると詳細が見える」体験の土台で、
T011(changed files)/ T012(diff)はこのパネルの中に積んでいく。
GPUI のクリックイベント処理(`cx.listener` / `on_click`)をプロジェクトに導入する最初のチケット。

## スコープ

1. 選択状態:
   - `KagiApp` に `selected: Option<usize>`(行 index)を追加
   - commit 行クリックで選択(`on_click`)。同じ行を再クリックで解除してよい(任意)
   - 選択行はハイライト表示(背景色変更)
   - 選択時に `[kagi] selected: <short_id>` を stderr ログ
   - 実装ヒント: `uniform_list` の `cx.processor` クロージャは `(&mut KagiApp, Range, &mut Window, &mut Context<KagiApp>)` を受けるので、その `cx` から `cx.listener(...)` で行ごとの on_click ハンドラを作れる(T008 実装メモ参照)。行 div には既に `.id(ix)` がある
2. Detail Panel(右側、幅 360px 固定):
   - 選択なし → パネル非表示(リストが全幅)
   - 選択あり → 表示項目:
     - full SHA(折り返し可)
     - author: `name <email>` + 絶対日時(`YYYY-MM-DD HH:MM` UTC でよい。epoch から自前計算: civil date アルゴリズム。**新規 crate 禁止**)
     - committer(author と異なる場合のみ表示)
     - parents: short id の列(複数なら "merge of ..." と分かる見せ方)
     - full message(複数行。スクロールは不要、`overflow_hidden` で可)
   - パネル用の表示データは snapshot 取り込み時に事前構築(`CommitDetail` 構造体を `Vec` で保持。クリック時は index 参照のみ)
3. レイアウト: `flex_row` で「リスト(flex_1)+ パネル(w 360 固定)」。ヘッダ行は全幅のまま

## リスク

- `on_click` には行 div に `.id()` が必要(既にある)。`cx.listener` の move キャプチャで index を取り込む
- uniform_list の行は描画ごとに作り直されるので、選択状態は KagiApp(entity)側に持つこと(行側に持たない)
- 絶対日時の自前計算はうるう年処理を間違えやすい → Howard Hinnant の civil_from_days アルゴリズムをそのまま使う(doc comment に出典を書く)

### 使った gpui API

#### `cx.listener` シグネチャ (gpui 0.2.2)

ソース: `~/.cargo/registry/src/index.crates.io-1949cf8c6b5b557f/gpui-0.2.2/src/app/context.rs`

```rust
pub fn listener<E: ?Sized>(
    &self,
    f: impl Fn(&mut T, &E, &mut Window, &mut Context<T>) + 'static,
) -> impl Fn(&E, &mut Window, &mut App) + 'static
```

#### `on_click` シグネチャ (gpui 0.2.2)

ソース: `~/.cargo/registry/src/index.crates.io-1949cf8c6b5b557f/gpui-0.2.2/src/elements/div.rs`

```rust
fn on_click(mut self, listener: impl Fn(&ClickEvent, &mut Window, &mut App) + 'static) -> Self
```

要件: 行 div に `.id(ix)` が付いていること(既存)。`cx.listener` で `ClickEvent` 型を E として束縛。

#### 実際の呼び出しパターン

```rust
// render_rows 内 (cx は cx.processor から受け取る &mut Context<KagiApp>)
let click_handler = cx.listener(move |this, _event: &gpui::ClickEvent, _window, _cx| {
    this.select(ix);
});
div().id(ix).on_click(click_handler)
```

#### `when_some` の使い方

```rust
div()
    .when_some(detail, |el, d| el.child(render_detail_panel(d)))
```

`FluentBuilder::when_some` は `Option<T>` が Some のときだけクロージャを適用する。Detail Panel の条件分岐に使用。

### civil_from_days アルゴリズム

Howard Hinnant (2013) の verbatim Rust 移植。うるう年処理は算術のみで正確。
出典: http://howardhinnant.github.io/date_algorithms.html#civil_from_days

注意: テスト `non_leap_year_no_feb29` の day_number は Python の `datetime.date` で確認済み:
- `1900-02-28` = day `-25509` (1900 は 400 の倍数でないため閏年でない)
- `1900-03-01` = day `-25508`

### レイアウト変更点

- T008/T009 では `uniform_list` が直接 `flex_1` で全幅を使っていた
- T010 では `div().flex().flex_row().flex_1()` のボディ行を追加し、その中に `uniform_list(flex_1)` + `when_some(detail, panel(w360))` を配置
- ヘッダ行は `w_full` のまま変更なし

### 注意点

1. `cx.processor` のクロージャ引数は `&mut Context<KagiApp>` なので、そこから `cx.listener(...)` を呼べる
2. `uniform_list` の行は描画ごとに再生成されるため選択状態は `KagiApp.selected` (entity 側) に持つ
3. `render_detail_panel` 内の mutable `let mut panel = div()...` パターン: committer の条件付き追加に使用
