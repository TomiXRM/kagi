# T010: commit selection で metadata panel を表示する

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T008, T009

## 背景

requirements M-05 の前半。「commit を選択すると詳細が見える」体験の土台で、
T011(changed files)/ T012(diff)はこのパネルの中に積んでいく。
GPUI のクリックイベント処理(`cx.listener` / `on_click`)をプロジェクトに導入する最初のチケット。

## スコープ

1. 選択状態:
   - `KagiApp` に `selected: Option<usize>`(行 index)を追加
   - commit 行クリックで選択(`on_click`)。同じ行を再クリックで解除してよい(任意)
   - 選択行はハイライト表示(背景色変更)
   - 選択時に `[kagi] selected: <short_id>` を stderr ログ
   - 実装ヒント: `uniform_list` の `cx.processor` クロージャは `(&mut KagiApp, Range, &mut Window, &mut Context<KagiApp>)` を受けるので、その `cx` から `cx.listener(...)` で行ごとの on_click ハンドラを作れる(T008 実装メモ参照)。行 div には既に `.id(ix)` がある
2. Detail Panel(右側、幅 360px 固定):
   - 選択なし → パネル非表示(リストが全幅)
   - 選択あり → 表示項目:
     - full SHA(折り返し可)
     - author: `name <email>` + 絶対日時(`YYYY-MM-DD HH:MM` UTC でよい。epoch から自前計算: civil date アルゴリズム。**新規 crate 禁止**)
     - committer(author と異なる場合のみ表示)
     - parents: short id の列(複数なら "merge of ..." と分かる見せ方)
     - full message(複数行。スクロールは不要、`overflow_hidden` で可)
   - パネル用の表示データは snapshot 取り込み時に事前構築(`CommitDetail` 構造体を `Vec` で保持。クリック時は index 参照のみ)
3. レイアウト: `flex_row` で「リスト(flex_1)+ パネル(w 360 固定)」。ヘッダ行は全幅のまま

## リスク

- `on_click` には行 div に `.id()` が必要(既にある)。`cx.listener` の move キャプチャで index を取り込む
- uniform_list の行は描画ごとに作り直されるので、選択状態は KagiApp(entity)側に持つこと(行側に持たない)
- 絶対日時の自前計算はうるう年処理を間違えやすい → Howard Hinnant の civil_from_days アルゴリズムをそのまま使う(doc comment に出典を書く)
