# T011: changed files list を表示する

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T010

## 背景

requirements M-05 の後半。「commit を選択すると metadata と **changed files** が見える」を完成させる。
ここで作る diff バックエンドの土台(first-parent との tree diff)は T012(file diff viewer)でそのまま拡張する。

## スコープ

1. backend(`src/git/diff.rs` 新規、lib 側):
   - `pub fn commit_changed_files(repo: &Repository, id: &CommitId) -> Result<Vec<FileStatus>, GitError>`
   - 比較対象: **commit の tree vs first parent の tree**(root commit は empty tree と比較)
   - `diff_tree_to_tree` + rename 検出(`diff.find_similar(&mut DiffFindOptions::new().renames(true))`)
   - delta の status → 既存 `ChangeKind`(Added / Modified / Deleted / Renamed{from} / TypeChange)に変換。
     その他の delta status(Copied 等)は Modified に丸めてよい(メモに書く)
   - 既存の `FileStatus` / `ChangeKind`(status.rs)を再利用。新型を作らない
2. integration test(`tests/diff_test.rs`):
   - tempdir + git CLI で組み立て、以下を assert:
     - added / modified / deleted / renamed(`git mv` 相当 + 類似度検出)の各 1 ファイル
     - root commit(全ファイル Added)
     - merge commit(first parent との diff であること: second parent 由来の変更が一覧に出る)
3. UI(Detail Panel 拡張):
   - `KagiApp` に `repo_path: Option<PathBuf>` を保持し、**選択時にオンデマンドで** `open_repository` 相当 + `commit_changed_files` を呼ぶ(全 commit 分の事前計算は禁止: 10k commits で遅すぎる)
   - 結果は `HashMap<usize, Vec<...>>` 等でキャッシュ(同じ commit の再選択で再計算しない)
   - パネルの Message の下に `Changed files (N)` セクション: 各行は change kind の1文字バッジ(A=green / M=yellow / D=red / R=blue / T=gray)+ パス(長い場合は先頭を `…` で省略してよい)
   - 表示は最大 100 件で truncate し、超過分は `… and N more` と出す
   - diff 取得に失敗してもクラッシュせず `（diff unavailable）` 表示
   - 選択時ログに `[kagi] changed files: N` を追加(headless 検証用。`KAGI_SELECT_FIRST=1` で出ること)
4. UI 側の取得は同期で可(MVP。バックグラウンド化は T017 以降の課題としてメモ)

## リスク

- rename 検出は `find_similar` を呼ばないと Added+Deleted に分かれる(テストの renamed ケースで検出可能)
- root commit の empty tree 比較: `diff_tree_to_tree(None, Some(&tree), ...)` で `old_tree=None` を渡せる
- merge commit の diff は「first parent と比較」が GUI の通例(git show と同じ)。combined diff はやらない
- 選択のたびに Repository::open するコスト: MVP では許容(ローカル SSD で ms オーダー)。気になる場合もリファクタしない(T017 でワーカー化を検討)
