- Status: in-progress
- 担当: Sonnet subagent
- 依存: T011

## 背景

requirements M-06。changed files のファイルをクリックすると unified diff が読める。
読み取り系 UI の最後のピース。これが終わると T013 から操作系(plan パイプライン)に入る。

## スコープ

1. backend(`src/git/diff.rs` 拡張):
   - architecture.md §3 の Diff モデルを導入:
     - `FileDiff { old_path: Option<PathBuf>, new_path: Option<PathBuf>, change: ChangeKind, hunks: Vec<Hunk>, is_binary: bool }`
     - `Hunk { old_range: (u32, u32), new_range: (u32, u32), lines: Vec<DiffLine> }`
     - `DiffLine { kind: DiffLineKind (Context|Added|Removed), content: String, old_lineno: Option<u32>, new_lineno: Option<u32> }`
   - `pub fn commit_file_diff(repo: &Repository, id: &CommitId, path: &Path) -> Result<FileDiff, GitError>`
   - 実装: T011 と同じ first-parent tree diff を `DiffOptions::pathspec(path)` で絞り、
     `git2::Patch::from_diff(&diff, 0)` から hunks / lines を読む(`Patch::hunk()` / `Patch::line_in_hunk()`)
   - rename されたファイルも引けるよう、pathspec には new path を渡す(find_similar 後の delta の new_file 基準)。
     困難なら rename ファイルは Modified 扱いの diff でよい(メモに書く)
   - binary は `delta.flags().is_binary()` 等で検出し hunks 空 + is_binary=true
   - line content は lossy UTF-8(panic しない)
2. integration test(`tests/diff_test.rs` に追記 or 新ファイル。既存テストの変更は禁止、追加のみ):
   - modified ファイル: hunk の old/new range と Added/Removed/Context 行の内容を `git show` の出力と矛盾しないことを assert(行数・内容の一致でよい)
   - added ファイル: 全行 Added
   - binary ファイル: is_binary=true
   - 日本語内容のファイルで panic しない
3. UI(Detail Panel の状態遷移):
   - `KagiApp` に `file_diff: Option<FileDiffView>` 的な状態を追加(選択 commit が変わったらクリア)
   - Changed files の行をクリック → panel が diff 表示モードに切り替わる(`cx.listener` + `cx.notify`)
   - diff 表示: 上部に `← back` 行(クリックで files 一覧に戻る)+ ファイル名、下に diff 本体
   - diff 本体は **`uniform_list`**(行数が多いため必須。id は "diff-list" 等、commit list とは別)
     - 行: `+` 緑背景 / `-` 赤背景 / context は無色。`text_sm`。長い行は overflow_hidden
     - hunk ヘッダ行(`@@ -a,b +c,d @@`)は青系で区切り表示
   - binary は「Binary file (no diff)」表示
   - panel 幅は diff 表示モードのときだけ 560px に広げてよい(任意)
4. headless 検証: `KAGI_OPEN_FIRST_FILE=1`(KAGI_SELECT_FIRST と併用)で最初の changed file の diff を自動で開き、
   `[kagi] diff: <path> hunks=N (+A -R)` をログ出力

## リスク

- `Patch::from_diff(&diff, idx)` の idx は pathspec で絞れば 0 だが、delta が複数残る場合がある(ディレクトリ pathspec 等)→ new_file().path() が一致する delta を探す方が堅い
- 行の `origin()` は '+', '-', ' ' 以外('=', '>', '<' 等の EOF newline マーカー)もある → Context 扱いに丸める
- uniform_list を同一 view に2つ置く場合、ElementId の衝突に注意(別 id を付ける)
- 文字列 truncate は chars() ベース(バイトスライス禁止)

## 実装メモ

### 使用した git2 API (Patch 周り)

- `git2::Patch::from_diff(&diff, idx) -> Result<Option<Patch<'_>>, Error>`
  - `None` が返るときはバイナリまたは変更なし。バイナリ検出の主なシグナルとして使用。
- `Patch::num_hunks() -> usize`
- `Patch::hunk(hunk_idx) -> Result<(DiffHunk<'a>, usize), Error>`
  - 第2戻り値はそのhunk内の行数。
- `DiffHunk::old_start() -> u32` / `old_lines() -> u32` / `new_start() -> u32` / `new_lines() -> u32`
- `Patch::line_in_hunk(hunk_idx, line_of_hunk) -> Result<DiffLine<'a>, Error>`
- `DiffLine::origin() -> char` — `'+'`/`'-'`/`' '`/`'='`/`'>'`/`'<'`/`'F'`/`'H'`/`'B'`
- `DiffLine::content() -> &[u8]` — バイトスライス、`String::from_utf8_lossy` で変換
- `DiffLine::old_lineno() -> Option<u32>` / `new_lineno() -> Option<u32>`
- `DiffFile::is_binary() -> bool` — フラグ `GIT_DIFF_FLAG_BINARY`。`diff_tree_to_tree` 後はコンテンツ検査前なので未セットのことがある。Patchが`None`の方が確実。
- `Diff::deltas() -> Deltas<'_>` — `num_deltas()` はなく `.count()` で数える
- `Diff::get_delta(i) -> Option<DiffDelta<'_>>`

### binary 検出の設計判断

`DiffFile::is_binary()` は `diff_tree_to_tree` 直後では未設定のことがある（コンテンツ検査が遅延されるため）。そのため、`Patch::from_diff` が `None` を返した場合に `is_binary: true` とする方式を採用した。`None` は「変更なし」にもなりうるが、`diff_tree_to_tree` + pathspec では変更ありの delta のみ返ってくるため、実用上 `None` = バイナリとして問題ない。

### rename pathspec

Ticket に記載の通り、pathspec には new path を渡す。rename の場合、`find_similar` 後は new_file().path() が新パスになっているため、UI 側から渡された path でそのまま一致する。

### EOF marker 行の処理

`DiffLine::origin()` の `'='`(Context EOF) → `Context`、`'>'`(Add EOF) → `Added`、`'<'`(Remove EOF) → `Removed` に折りたたむ。

### UI 状態遷移

- `KagiApp.file_diff_view: Option<FileDiffView>` を追加。
- commit 行のクリック → `select()` 内で `file_diff_view = None` にリセット。
- ファイル行のクリック → `open_file_diff(file_index)` → diff 取得 + `FileDiffView` をセット。
- "← back" クリック → `close_file_diff()` → `file_diff_view = None`。
- diff パネルは別 id `"diff-list"` の `uniform_list` を使用（commit-list との id 衝突回避）。
- diff パネル幅: 560px（通常の detail panel は 360px）。
