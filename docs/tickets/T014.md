# T014: create branch を実装する

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T013

## 背景

requirements M-08。T013 で敷いた plan パイプラインに2つ目の操作を乗せる。
create branch は **Safe クラス**(ADR-0004): working tree も既存 commit も変更しない。
このチケットで **テキスト入力**(branch 名)を初導入する。

## スコープ

### 1. ドメイン(`src/git/ops.rs` 追記)

- `pub fn plan_create_branch(repo: &Repository, name: &str, at: &CommitId) -> Result<OperationPlan, GitError>`
  - blocker:
    - 同名 branch が既に存在
    - branch 名が不正(`git2::Reference::is_valid_name(&format!("refs/heads/{name}"))` で検証)
    - 名前が空 / 起点 commit が存在しない
  - warning: なし(Safe クラス)
  - predicted: `head` は変わらない(checkout はしない)。`新 branch '<name>' @ <short sha>` が増える旨を title/predicted に反映
  - recovery: `git branch -d <name>`(未 merge でも作成直後なら -d で消せる旨)
- `pub fn execute_create_branch(repo: &Repository, name: &str, at: &CommitId) -> Result<(), GitError>`
  - `repo.branch(name, &commit, false)`(force=false 固定)
  - **checkout はしない**(branch 作成のみ。GitKraken と違う点だがスコープを絞る)
- preflight は T013 の `preflight_check` を再利用

### 2. UI

- Detail Panel(commit 選択時)に `+ Create branch here` ボタンを追加 → create-branch モーダル
- モーダル: branch 名の**1行テキスト入力** + plan 表示(入力が変わるたびに plan を再生成して blocker を live 表示)+ `[Cancel]` `[Create]`(blocker 時は Create 非表示)
  - テキスト入力は gpui 0.2.2 の `examples/input.rs`(registry 内)を参考に最小実装
    (focus + キー入力。IME 対応は不要、ASCII 入力が動けば可)
  - 入力実装が過大になる場合は、`KeyDownEvent` を拾う簡易実装(英数字 / `-` `_` `/` `.` + backspace のみ)で可。メモに明記
- 実行フロー: confirm → preflight → execute → snapshot 再取得 → reload(badge に新 branch が出る)
- ログ: `plan: create-branch <name> blockers=N` / `executed: create-branch <name> @ <short>` / `verified: branch exists`

### 3. headless 検証 env

- `KAGI_CREATE_BRANCH=<name>`: 起動後、HEAD commit を起点に plan 生成 + ログ。`KAGI_AUTO_CONFIRM=1` 併用で実行まで

### 4. integration test(`tests/ops_test.rs` 追記。既存テストの変更禁止)

- 正常系: branch が指定 commit を指して作成される。HEAD・working tree が変わらない
- 同名既存 → blocker / 不正名(空白入り, `..` 入り, 先頭 `-`)→ blocker / 空文字 → blocker
- force=false: 既存 branch を上書きしないことを execute レベルでも確認(blocker を無視して直接 execute を呼んでも失敗する)

## リスク

- gpui のテキスト入力は本格実装(IME, selection)が重い → 簡易 KeyDown 実装で割り切る(このプロダクトの主役ではない)
- branch 名 validation は git2 の `Reference::is_valid_name` が正(自前正規表現で頑張らない)
- `repo.branch()` の force 引数は必ず false リテラル(変数化しない)

## 実装メモ

### テキスト入力の実装方式

**KeyDownEvent ベースの簡易実装**を採用した。

理由:
- `gpui 0.2.2` の `EntityInputHandler` (IME 対応フル実装) は `text_for_range` / `replace_text_in_range` 等の複雑なトレイトを要求し、実装コストが大きい。
- このプロダクトで branch 名入力は ASCII のみで十分（IME 不要）。
- `cx.listener(|this, event: &KeyDownEvent, ..|)` で十分なキャプチャが可能。

受付文字: ASCII 英数字 `a-z A-Z 0-9` および `-` `_` `/` `.` と `backspace`。
modifier キー（cmd/ctrl/alt）付きは無視する。

### blocker 追加: 先頭 `-`

`git2::Reference::is_valid_name("refs/heads/-bad-name")` は `true` を返す（git 仕様上 ref 名として valid だが、コマンドライン引数と混同される可能性がある）。チケット仕様に基づき `name.starts_with('-')` を追加の blocker チェックとして実装した。
