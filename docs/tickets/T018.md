# T018: changed files をファイルツリー表示にする(ユーザー要望)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T011, T012
- 発端: ユーザー要望「右側の diff ペインの変更ファイルを、リストではなくファイルツリーで表示してほしい」

## 背景

T011 の changed files はフラットなパス一覧。ネストの深い repo では見通しが悪い。
GitKraken / VSCode と同様の**ディレクトリツリー表示**に変更する。

## スコープ

1. ツリー構築の pure ロジック(`src/ui/file_tree.rs` 新規):
   ```rust
   pub enum TreeRow {
       Dir  { depth: usize, name: SharedString },
       File { depth: usize, name: SharedString, file_index: usize, change: ChangeKind },
   }
   pub fn build_file_tree(files: &[FileStatus]) -> Vec<TreeRow>
   ```
   - パスを `/` で分割してディレクトリツリーを構築し、**深さ優先・各階層でディレクトリ先行 + 名前順**で flatten
   - **単一子ディレクトリの連結圧縮**(`a/` の中が `b/` だけなら `a/b` と1行にする。VSCode 方式)
   - `file_index` は入力 `files` の index(クリック時に T012 の `open_file_diff(file_index)` へそのまま渡せる)
   - rename(`Renamed { from }`)はファイル名の後に `← <from>` を補足表示できるよう name に含めてよい
   - **`#[cfg(test)]` unit test 必須**: フラット / ネスト / 単一子圧縮 / dir・file 混在ソート / 日本語パス の5ケース以上
2. UI(`src/ui/mod.rs` の Changed files セクション置き換え):
   - フラット一覧 → `build_file_tree` の行を描画(depth × 12px の indent)
   - Dir 行: フォルダ風の薄色表示(クリック不可で可。折りたたみは today やらない)
   - File 行: 従来どおり change kind バッジ(A/M/D/R/T 色分け)+ ファイル名(パス全体ではなく末端名)、
     クリックで diff を開く(既存の `open_file_diff`)
   - truncate ポリシー(最大100件)とログ(`changed files: N`)は維持
3. 既存の headless 検証(`KAGI_SELECT_FIRST` / `KAGI_OPEN_FIRST_FILE`)が引き続き動くこと
   (`file_index` のマッピングを壊さない)

## 実装メモ

### 変更ファイル

- `src/ui/file_tree.rs` (新規): ツリー構築純粋ロジック + unit tests 8ケース
- `src/ui/mod.rs`:
  - `pub mod file_tree;` 追加
  - `render_detail_panel` の Changed files セクションをツリー描画に置き換え
  - `select()` に `KAGI_SELECT_FIRST=1` 時の `[kagi] tree:` ログ出力を追加

### ツリー構築アルゴリズム

1. `PathBuf::components()` でパスを分解 (文字列 split なし)
2. n-ary `DirNode` ツリーに挿入 (`BTreeMap` で自動ソート)
3. 単一子ディレクトリ圧縮: `dirs.len()==1 && files.is_empty()` の場合に子ディレクトリ名を `/` で連結し再帰 (`a/b/c` のような形に)
4. 深さ優先でフラット化: 各レベルでディレクトリ先行、ファイル名順

### truncate ポリシー

入力 files を先頭 100 件に切ってからツリー構築 (`file_index` は元の slice の index を保持)
