## スコープ

### 1. グラフエッジの直角 + 丸角化(`src/ui/graph_view.rs`)

現在: IntoNode / OutOfNode の lane またぎが**斜めの直線**。
変更後: **縦線 → 丸角コーナー → 横線**(GitKraken / gitk 風)。

- `CORNER_R ≈ 6.0`(px)。lane 間隔や行高に対して大きすぎる場合は `min(CORNER_R, |dx|/2, 利用可能な縦距離/2)` にクランプ
- **IntoNode**(from_lane j → node lane、j ≠ node):
  `(x_j, 0)` から縦に `(x_j, mid_y - R)` → 丸角 → 横線 `(x_node ± R, mid_y)` → node 中心へ
- **OutOfNode**(node lane → to_lane j、j ≠ node):
  node 中心から横線 `(x_j ∓ R, mid_y)` → 丸角 → `(x_j, mid_y + R)` から縦に `(x_j, row_h)`
- from == to のケース(直進)は従来どおり縦線
- 丸角は PathBuilder の bezier 系 API(`curve_to` / `cubic_bezier_to` 等。
  **registry の examples/painting.rs と src/path_builder.rs で実 API を確認**)。
  bezier がなければ小さい line セグメント数本での近似でも可(メモに明記)
- 線幅・色規則(IntoNode=from色 / OutOfNode=to色)は変更しない

## 実装メモ

### 丸角エッジ実装方式

`PathBuilder::curve_to(to, ctrl)` (二次ベジェ) を採用。

シグネチャ: `curve_to(&mut self, to: Point<Pixels>, ctrl: Point<Pixels>)` → 内部で `quadratic_bezier_to(ctrl, to)` を呼ぶ。

コーナー形状:
- **IntoNode** (from_lane → node_lane): 垂直線 `(x_from, y_top)` → `(x_from, mid_y-R)` → 二次ベジェ `ctrl=(x_from, mid_y), end=(x_from±R, mid_y)` → 水平線 `→ (x_node, mid_y)`
- **OutOfNode** (node_lane → to_lane): 水平線 `(x_node, mid_y)` → `(x_to∓R, mid_y)` → 二次ベジェ `ctrl=(x_to, mid_y), end=(x_to, mid_y+R)` → 垂直線 `→ (x_to, y_bot)`
- **同一レーン直進** (from == to): 従来の縦線を維持

R クランプ: `R = min(CORNER_R=6.0, |dx|/2, avail_vertical/2)` — R が 0.5 未満または dx が 0.5 未満のとき直線フォールバック。

### アバター実装

- ハッシュ: **FNV-1a 32-bit** (自前実装。offset_basis=2166136261, prime=16777619)
- パレット: hue を 12 分割 (`bucket = hash % 12, hue = bucket / 12.0`)、saturation=0.70、lightness=0.60、alpha=1.0
- イニシャル: `chars().next().to_uppercase().collect::<String>()`、空文字は `"?"`
  - grapheme cluster 境界は無視 (`chars()` ベース)。日本語/CJK はそのまま1文字。ドイツ語 ß → "SS" (char::to_uppercase の標準動作)
- 表示: `div().w(px(18.)).h(px(18.)).rounded_full().bg(Hsla)` に白テキスト `text_xs()`。graph 領域と SHA の間、`mr(px(4.))` のマージン付き。行高 24px 維持。
