# T021: commit 行レイアウトを GitKraken 順に変更(ユーザー要望)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T020
- 発端: ユーザー指摘「GitKraken は [branch list][[branch/tag][graph][commit message]][commit details<hash>] の並び」

## 背景

現状の commit 行: `graph | avatar | SHA | badges | summary | author | date`。
GitKraken の並びに合わせ、**バッジをグラフの左**に置き、**SHA カラムを行から外す**(hash は選択時の Detail Panel にある)。

## スコープ

1. commit 行の並びを変更(`src/ui/mod.rs` の `render_rows`):
   ```
   [badges 列(固定幅・右寄せ)] [graph lanes] [avatar] [summary(flex_1)] [author] [date]
   ```
   - badges 列: 固定幅 **150px**、`flex_row` 右寄せ(`justify_end`)、`overflow_hidden`。
     バッジが多い場合は先頭側(HEAD/branch 優先)を残して溢れは clip(truncate 24chars は維持)
   - バッジなしの行は空の 150px 領域(グラフの開始位置を全行で揃える — これが GitKraken の見た目の肝)
   - SHA(short_id)のカラムは**削除**(CommitRow の short_id フィールド自体は Detail Panel 連携・ログで使うので残す)
   - avatar / summary / author / date は従来どおり(グラフの右に avatar → message)
2. バッジ内の優先順位: HeadBranch → Branch → Tag → Remote の順に表示(右寄せなので「グラフに近い側」に重要なものが来るよう並べる)
3. 既存 headless 検証・テストに回帰なし(表示順の変更のみ。データ構造・クリック挙動は不変)
