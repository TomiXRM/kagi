- Status: in-progress
- 担当: Sonnet subagent
- 依存: T019, T021
- 発端: ユーザー報告(実 repo でのテスト)
  1. 「コミット詳細ペインが縦スクロールできず、file diff が映らないことがある」
  2. 「文字の折り返しを無くしてほしい。まだテキストオーバーフローがある」

## 背景

実 repo では changed files が多く message も長いため、詳細ペインの内容が縦に溢れて diff 領域が画面外に押し出される。
また T019 の ZWSP 折り返し方式は「折り返さず1行で切る」方針に転換する(ユーザー要望)。

## スコープ

### 1. 詳細ペインの縦スクロール

- **metadata 表示モード**(SHA / Author / Parents / Message / Changed files tree)全体を縦スクロール可能にする
  - gpui 0.2.2 のスクロール API を registry の `examples/scrollable.rs` と `src/styled.rs` で確認
    (`.id(..)` 付き要素の `overflow_y_scroll()` / `overflow_scroll()` 等。uniform_list 以外の任意コンテンツのスクロール方法)
  - パネルは `flex_col` で「スクロール領域(flex_1)」構成にし、内容が短いときは従来どおり
- **diff 表示モード**: `← back` ヘッダ(固定)+ diff の uniform_list(flex_1)が**必ずパネル高さに収まって**スクロールすることを確認・修正
  (現状 file diff が映らないことがある報告の根本原因をここで特定して直す。原因と修正内容をメモに書くこと)

### 2. 折り返し廃止(ZWSP 方式の撤去)

- `soft_wrap` の使用を全廃し、**1行 + ellipsis** に統一する:
  - SHA: 1行・`truncate()`(gpui の overflow_hidden + nowrap + ellipsis 複合)。値の全体は選択コピーできないが MVP 許容
  - Author / Committer 行: 1行・truncate
  - Message: **改行(\n)はそのまま行として表示**し、各行を nowrap + truncate(長い行は…で切る)。
    人工的な折り返しはしない
  - モーダルの warnings / blockers / recovery / error: 各項目 nowrap + truncate はやりすぎ(内容が読めなくなる)。
    ここは「**\n で明示的に分割した複数行**」+ 各行 truncate にする(soft_wrap は使わない。文言を短い行に分ける)
  - `soft_wrap` 関数と関連テストは削除(または deprecated コメント付きで未使用化。どちらにしたかメモに書く)
- バッジ truncate(24chars)・ヘッダ overflow_hidden 等の既存 clip は維持

### 3. 残オーバーフローの総点検

- ストレス repo(長い日本語 message / 長 ASCII トークン / 長パス / 長 branch 名 / 長 email)+
  実形状に近い repo(多数 changed files)で全画面状態(通常 / 選択 / diff / 各モーダル)を一巡し、
  はみ出しゼロを確認。チェックした画面の一覧をメモに書く
- 行系(commit 行 / ツリー行 / diff 行 / sidebar 行)はすべて nowrap + 親で clip されることを確認

## 実装メモ

### file diff が映らない根本原因と修正

**根本原因**:  
`render_diff_panel` の `flex_col` コンテナ内で `uniform_list` に `.flex_1().h_full()` を付与していたが、
flex アルゴリズムでは子要素の `min-height` がデフォルト `auto`(= コンテンツ自然サイズ)のため、
`"← back"` ヘッダの高さが確定している状態でも `uniform_list` の flex-shrink が効かず、
パネル高さを超えてオーバーフローしてしまっていた。
結果として `uniform_list` はパネル外部に押し出され、diff 行が視覚的に見えなかった。

**修正**:  
`uniform_list` から `.h_full()` を削除し、`.flex_1().min_h(px(0.))` に変更。
`min_h(px(0.))` を付与することで flex 子要素が親高さまで縮むことが保証される。
`"← back"` ヘッダには `.flex_shrink_0()` を追加して固定高さを確定させた。

### スクロール API シグネチャ

```
// StatefulInteractiveElement (`.id(..)` が必要)
fn overflow_y_scroll(mut self) -> Self
fn overflow_scroll(mut self) -> Self   // x+y 両方

// Styled (`.id(..)` 不要)
fn truncate(mut self) -> Self  // overflow_hidden + whitespace_nowrap + text_ellipsis の複合
```

`overflow_y_scroll()` はスクロール状態を要素 ID で追跡するため `.id("detail-scroll")` が必須。
`min_h(px(0.))` は flex コンテナ内でスクロール領域を正しく縮めるための必須セット。

### 変更ファイル

- `src/ui/detail_panel.rs` — `soft_wrap()` 関数・ZWSP 関連フィールド (`sha_display` 等) を全廃。
  `CommitDetail` は ZWSP なしの raw フィールドのみ保持。テストを T022 仕様に書き換え。
- `src/ui/mod.rs` — `soft_wrap` import 削除。`render_detail_panel` を縦スクロール化
  (`.id("detail-scroll").flex_1().min_h(px(0.)).overflow_y_scroll()`)。
  SHA/Author/Committer/Parents は `truncate()`、Message は `\n` 分割の行ごと `truncate()`。
  `render_diff_panel` の `uniform_list` を `.flex_1().min_h(px(0.))` に修正 + ヘッダに `flex_shrink_0()`。
  全モーダル (checkout/create-branch/stash-push/stash-apply/cherry-pick) の `soft_wrap()` 呼び出しを削除。
