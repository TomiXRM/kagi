# T022: 詳細ペインの縦スクロール + 折り返し廃止(ユーザー報告)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T019, T021
- 発端: ユーザー報告(実 repo でのテスト)
  1. 「コミット詳細ペインが縦スクロールできず、file diff が映らないことがある」
  2. 「文字の折り返しを無くしてほしい。まだテキストオーバーフローがある」

## 背景

実 repo では changed files が多く message も長いため、詳細ペインの内容が縦に溢れて diff 領域が画面外に押し出される。
また T019 の ZWSP 折り返し方式は「折り返さず1行で切る」方針に転換する(ユーザー要望)。

## スコープ

### 1. 詳細ペインの縦スクロール

- **metadata 表示モード**(SHA / Author / Parents / Message / Changed files tree)全体を縦スクロール可能にする
  - gpui 0.2.2 のスクロール API を registry の `examples/scrollable.rs` と `src/styled.rs` で確認
    (`.id(..)` 付き要素の `overflow_y_scroll()` / `overflow_scroll()` 等。uniform_list 以外の任意コンテンツのスクロール方法)
  - パネルは `flex_col` で「スクロール領域(flex_1)」構成にし、内容が短いときは従来どおり
- **diff 表示モード**: `← back` ヘッダ(固定)+ diff の uniform_list(flex_1)が**必ずパネル高さに収まって**スクロールすることを確認・修正
  (現状 file diff が映らないことがある報告の根本原因をここで特定して直す。原因と修正内容をメモに書くこと)

### 2. 折り返し廃止(ZWSP 方式の撤去)

- `soft_wrap` の使用を全廃し、**1行 + ellipsis** に統一する:
  - SHA: 1行・`truncate()`(gpui の overflow_hidden + nowrap + ellipsis 複合)。値の全体は選択コピーできないが MVP 許容
  - Author / Committer 行: 1行・truncate
  - Message: **改行(\n)はそのまま行として表示**し、各行を nowrap + truncate(長い行は…で切る)。
    人工的な折り返しはしない
  - モーダルの warnings / blockers / recovery / error: 各項目 nowrap + truncate はやりすぎ(内容が読めなくなる)。
    ここは「**\n で明示的に分割した複数行**」+ 各行 truncate にする(soft_wrap は使わない。文言を短い行に分ける)
  - `soft_wrap` 関数と関連テストは削除(または deprecated コメント付きで未使用化。どちらにしたかメモに書く)
- バッジ truncate(24chars)・ヘッダ overflow_hidden 等の既存 clip は維持

### 3. 残オーバーフローの総点検

- ストレス repo(長い日本語 message / 長 ASCII トークン / 長パス / 長 branch 名 / 長 email)+
  実形状に近い repo(多数 changed files)で全画面状態(通常 / 選択 / diff / 各モーダル)を一巡し、
  はみ出しゼロを確認。チェックした画面の一覧をメモに書く
- 行系(commit 行 / ツリー行 / diff 行 / sidebar 行)はすべて nowrap + 親で clip されることを確認
