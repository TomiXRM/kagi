# T023: ペインのリサイズ対応(ユーザー要望)

- Status: in-progress(T024 と並行: ファイル独立)
- 担当: Sonnet subagent
- 依存: T021, T022

## 背景

ユーザー要望「ペインのリサイズに対応してほしい」。
sidebar(現200px固定)と詳細ペイン(現360/560px固定)の幅をドラッグで変更できるようにする。

## スコープ

1. `KagiApp` に `sidebar_width: f32` / `panel_width: f32` を追加(初期値は現行値。永続化は later)
2. **ドラッグ可能なディバイダ**を2本:
   - sidebar | メイン の間
   - メイン | 詳細ペイン の間(詳細ペイン表示時のみ)
   - 見た目: 幅 4px の縦バー(hover で色が変わる)。gpui のマウスイベント
     (`on_mouse_down` / `on_mouse_move` / `on_mouse_up`、`registry の examples/drag_drop.rs` 等で API 確認)で実装
   - ドラッグ中: `KagiApp` に `dragging: Option<DividerKind>` を持ち、window の mouse move で幅を更新 + `cx.notify()`
3. クランプ: sidebar 120〜400px / 詳細ペイン 240〜800px(ウィンドウ幅の8割を超えない)
4. diff 表示モードの 560px 特例は廃止し、`panel_width` に一本化(リサイズできるので不要)
5. グラフ・リスト・ツリーは flex なので追従するはず(回帰確認)
