## 背景

Commit Panel(T025)に必要な4機能をドメイン層に実装する:
file 単位の stage / unstage、workdir・index の diff、commit 作成。
stage/unstage は index のみを変更(working tree のファイル内容には触れない)= Safe クラス。
commit 作成は plan パイプラインに乗せる。hunk/line 単位は v0.2/v0.3(やらない)。

## スコープ(`src/git/staging.rs` 新規 + ops.rs 追記)

### 1. stage / unstage(file 単位)

- `pub fn stage_file(repo: &Repository, path: &Path) -> Result<(), GitError>`
  - 通常: `index.add_path(path)` → `index.write()`
  - **workdir で削除されたファイル**: `index.remove_path(path)`(存在チェックで分岐)
- `pub fn unstage_file(repo: &Repository, path: &Path) -> Result<(), GitError>`
  - `git reset HEAD -- <path>` 相当: HEAD tree に path があれば index entry を HEAD の内容に戻し、
    なければ index から remove(`repo.reset_default(Some(&head_obj), [path])` が使えるか docs.rs で確認。
    使えなければ index 直接操作)
  - unborn HEAD(初 commit 前)の場合は index から remove
- どちらも **working tree のファイル内容は不変**(テストで assert)

### 2. workdir / index の diff(Commit Panel の diff 表示用)

- `pub fn unstaged_file_diff(repo: &Repository, path: &Path) -> Result<FileDiff, GitError>`
  — index ↔ workdir(`diff_index_to_workdir` + pathspec。untracked は include_untracked + show_untracked_content
  で全行 Added になることを確認)
- `pub fn staged_file_diff(repo: &Repository, path: &Path) -> Result<FileDiff, GitError>`
  — HEAD tree ↔ index(`diff_tree_to_index`。unborn は old_tree=None)
- 既存の hunk/line 変換(T012 の Patch 処理)を共通化して再利用(commit_file_diff と重複実装しない。
  共通ヘルパー抽出は diff.rs 内のリファクタとして可)

### 3. commit 作成(plan パイプライン)

- `pub fn plan_commit(repo: &mut Repository, message: &str) -> Result<OperationPlan, GitError>`
  - blocker: staged が空 / message が空(trim 後)/ conflict 状態
  - warning: **unstaged / untracked の変更が残る**(「N file(s) はこの commit に含まれません」)
  - predicted: HEAD branch に新 commit(summary 表示)。preview_files = staged 一覧
  - recovery: 直後なら `git commit --amend` で message 修正可、`git revert` の案内(reset は出さない)
- `pub fn execute_commit(repo: &Repository, message: &str) -> Result<CommitId, GitError>`
  - index から write_tree → `repo.commit(Some("HEAD"), &sig, &sig, message, &tree, &[head_commit])`
    (sig は repo config。unborn HEAD は parents=[] で初 commit)
  - 実行後も unstaged 変更はそのまま残る(テストで assert)

### 4. integration test(`tests/staging_test.rs` 新規)

- stage: modified / untracked(new)/ deleted の3種 → `git status --porcelain` 突き合わせで INDEX 側に移ること + WT 内容不変
- unstage: 上記の逆方向 + 「HEAD にないファイル(new)を unstage」→ untracked に戻る
- unborn repo での stage → commit(初 commit)
- diff: unstaged_file_diff が WT の変更を、staged_file_diff が index の変更を返す(partial stage:
  同一ファイルを stage 後さらに WT 編集 → 両 diff が異なることを確認)
- plan_commit: blocker(空 staged / 空 message)・warning(unstaged 残り)・execute 後に staged が空 + 新 commit + unstaged 残存
- **全テスト tempdir のみ**
