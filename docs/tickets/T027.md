# T027: Commit Panel の Unstaged/Staged を 1:1 の独立スクロールボックスに(ユーザー要望)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T025, T026

## 背景

ユーザー要望「GitKraken では stage と unstage のペインが 1:1 のサイズで取られ、それぞれ独立してスクロールできる。独立した箱にしたい」。
現状は2セクションが内容量に応じて伸びる積み上げ式で、ファイルが多いと片方が画面を占有する。

## スコープ

1. Commit Panel の縦構成を以下に変更:
   ```
   ┌──────────────────────────────┐
   │ Unstaged (N)     [flat|tree] │ ← ヘッダ固定(flex_shrink_0)
   │ ┌─ scrollable box ─────────┐ │
   │ │ ...files...              │ │ ← 箱1
   │ └──────────────────────────┘ │
   │ Staged (M)                   │ ← ヘッダ固定
   │ ┌─ scrollable box ─────────┐ │
   │ │ ...files...              │ │ ← 箱2(箱1と常に同じ高さ = 1:1)
   │ └──────────────────────────┘ │
   │ ── diff viewer(従来) ──────  │
   │ ── message + commit(従来) ─  │
   └──────────────────────────────┘
   ```
   - 箱1・箱2はそれぞれ `flex_1` + `min_h(px(0.))` + `.id(..).overflow_y_scroll()`(T022 の定石)で
     **常に同じ高さ・独立スクロール**
   - 2つの箱を包むコンテナと diff 領域の高さ配分: ファイル領域(箱2つ)が概ねパネルの 40〜50%、
     diff が残り(diff 未選択時はファイル領域を広く使ってよい — 実装しやすい方で。メモに書く)
   - ファイルが少なく箱が余る場合は空白でよい(GitKraken と同じ)
2. flat/tree 切替・Stage/Unstage ボタン・選択 → diff・件数表示は現行機能を維持(回帰なし)
3. 各箱に薄い枠(`border_1` + BG_SURFACE 系)を付けて「独立した箱」に見えるようにする

## リスク

## 実装メモ

### 変更ファイル
- `src/ui/mod.rs` — `render_commit_panel` 関数のみ変更

### 高さ配分の設計
ファイル領域(cp-files-container)と diff 領域(cp-diff-area)をどちらも `flex_1 + min_h(px(0.))` にすることで **1:1 の等分** とした。
フッター(Commit message + ボタン)は `flex_shrink_0` で固定。

```
┌── パネル全体 (flex_col, h_full) ──────────────────────────┐
│ "Commit Panel" ヘッダ (flex_shrink_0)                      │
│ ┌── cp-files-container (flex_1, min_h(0)) ──────────────┐ │
│ │ Unstaged (N)  [flat|tree]  ← ヘッダ固定 (flex_shrink_0)│ │
│ │ ┌── cp-unstaged-scroll (flex_1, min_h(0), border) ──┐ │ │
│ │ │ ...ファイル行...                                    │ │ │
│ │ └────────────────────────────────────────────────────┘ │ │
│ │ Staged (M)                 ← ヘッダ固定 (flex_shrink_0)│ │
│ │ ┌── cp-staged-scroll   (flex_1, min_h(0), border) ──┐ │ │
│ │ │ ...ファイル行...                                    │ │ │
│ │ └────────────────────────────────────────────────────┘ │ │
│ └────────────────────────────────────────────────────────┘ │
│ ┌── cp-diff-area (flex_1, min_h(0)) ────────────────────┐ │
│ │ diff ヘッダ + diff viewer (uniform_list / placeholder)  │ │
│ └────────────────────────────────────────────────────────┘ │
│ フッター: commit message input + 警告 + Commit ボタン       │
│          (flex_shrink_0)                                    │
└────────────────────────────────────────────────────────────┘
```

ファイル領域とdiff領域が1:1になるため、ファイルが多い場合も画面の半分をファイル、半分をdiffが使う。
diff未選択時はプレースホルダー("Select a file to view diff")が表示され、ファイル領域と等しい高さを維持する。

### 枠の実装
各スクロールボックスに `.border_1().border_color(rgb(BG_SURFACE)).rounded_sm()` を付与した。
また `.mx_1().mb_px()` で内側に少しマージンを与えて視覚的に独立した箱に見えるようにした。

### 定石
`overflow_y_scroll()` には `.id()` が必須であるため、
- Unstaged: `.id("cp-unstaged-scroll")`
- Staged: `.id("cp-staged-scroll")`
と別々の id を割り当て、独立スクロールを保証した。
