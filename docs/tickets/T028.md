# T028: sidebar ブランチのクリック = グラフへジャンプ / ダブルクリック = checkout(ユーザー要望)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T013, T021

## 背景

ユーザー要望: 「ブランチリストのブランチをクリックしたら該当ブランチの tip commit にグラフのフォーカスを移したい(checkout はしない)。ダブルクリックで checkout したい」。
現状はシングルクリックで checkout plan モーダルが開く(T013)。これを GitKraken 流に変更する。

## スコープ

1. **シングルクリック**: 対象 branch の target CommitId → commit list の行 index を引き、
   - その行へ **uniform_list をスクロール**(`UniformListScrollHandle` を導入し
     `.track_scroll(..)` + `scroll_to_item(ix, ..)`。**gpui 0.2.2 の正確な API は
     `~/.cargo/registry/src/*/gpui-0.2.2/src/elements/uniform_list.rs` と examples/uniform_list.rs で確認**)
   - 同時にその行を **選択状態**にする(既存 `select()` — detail panel も開く)
   - WIP 行で index が +1 ずれている場合があるので CommitId → 行 index の対応は既存の対応箇所を踏襲
2. **ダブルクリック**: 既存の checkout plan モーダルを開く(`open_plan_modal`)
   - gpui の `ClickEvent` のクリック回数(`click_count` 相当)で判別(フィールド名は div.rs / ClickEvent 定義で確認)。
     1回目のクリックでジャンプ → 2回目でモーダル、の順で発火するのは許容(自然な挙動)
   - HEAD branch のダブルクリックは何もしない(従来どおり checkout 不可)
3. remote branch / tag / stash 行は今回対象外(branch のみ。余裕があれば remote branch のシングルクリックジャンプだけ追加可 — やった場合はメモに書く)
4. ログ: `[kagi] jump: <branch> -> row N` を出す(headless 検証用)
5. headless 検証 env: `KAGI_JUMP=<branch>` でシングルクリック相当を実行しログ確認
