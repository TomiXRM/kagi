# T030: commit list の列([branch][graph][message])を個別リサイズ可能に(ユーザー要望)

- Status: in-progress
- 担当: Sonnet subagent
- 依存: T021, T023

## 背景

ユーザー要望: 「GitKraken は [branch name][graph][commit message] をペインとして管理し、それぞれ横幅をリサイズできる。グラフを狭くしたら commit message ペインに隠れる(= クリップされる)」。
副次効果: graph ペインを広げれば lane_count が大きい repo(実例: 22 lanes)のグラフ全体が見える。
**MAX_LANES=8 の固定上限はこのチケットで廃止**し、「ペイン幅に入るだけ描く」に変える。

## スコープ

1. `KagiApp` に列幅を追加:
   - `badge_col_w: f32`(default 150.0、クランプ 60〜400)
   - `graph_col_w: f32`(default `min(lane_count, 8) * LANE_W` 相当の現行値、クランプ 28〜600)
   - message 列は従来どおり flex_1(残り全部)
2. **カラムヘッダ行**を commit list の上(WIP 行のさらに上)に新設:
   - `BRANCH / TAG | GRAPH | MESSAGE` のラベル(小さく muted)
   - ラベル間に **ドラッグハンドル2本**(T023 のディバイダと同じ見た目・4px)
   - ドラッグは既存の `DividerDrag` 機構を拡張(`DividerKind::BadgeCol` / `GraphCol` を追加)。
     **絶対座標方式**: 列の左端 x は `sidebar_width + 4 (+ 前列の幅 + 4)` で計算できる。
     `badge_col_w = cursor_x - 列左端`、`graph_col_w = cursor_x - graph列左端`(クランプ付き)
3. 行の描画を列幅に追従させる:
   - バッジ列: `w(px(badge_col_w))`(T028 までの 150 固定を置換。+N チップ方式は維持)
   - graph 列: `w(px(graph_col_w))` + **入るレーンだけ描く**:
     `graph_view` の `MAX_LANES` 定数依存をやめ、`graph_canvas` に「描画可能レーン数 =
     floor(width / LANE_W)」を引数で渡す(node が範囲外の行は node を端で省略してよい。
     線が graph 列の右へはみ出して message に重ならないこと = 範囲外レーンの edge はスキップ)
   - message / author / date は従来
4. カラムヘッダ行の高さは 20px 程度・`flex_shrink_0`。uniform_list の上に固定(リストと一緒にスクロールしない)
5. 列幅の永続化はしない(later)

## 実装メモ

### 変更ファイル

- `src/ui/graph_view.rs`
- `src/ui/mod.rs`

### 幅計算の設計（座標式）

**DividerKind 拡張**
```
BadgeCol — badge 列右端ハンドル
GraphCol — graph 列右端ハンドル
```

**絶対座標方式（drag-move リスナー内）**
```
badge_col_left = sidebar_width + 4       // sidebar divider = 4px
badge_col_w    = cursor_x - badge_col_left - 2   // clamp(60, 400)

graph_col_left = badge_col_left + badge_col_w + 4  // inner handle = 4px
graph_col_w    = cursor_x - graph_col_left - 2    // clamp(28, 600)
```

**visible_lanes（graph クリッピング）**
```
visible_lanes = floor(graph_col_w / LANE_W)
// graph_canvas に渡し、lane >= visible_lanes のエッジ/ノードをスキップ
```

**列幅デフォルト**
- badge_col_w: 150.0 px (clamped 60–400)
- graph_col_w: 8 * 14.0 = 112.0 px (clamped 28–600)
- message 列: flex_1 (残り全部)

### graph_view.rs の変更点

- `graph_canvas(node_lane, edges, visible_lanes)` に第3引数を追加
- `MAX_LANES` 定数はリファレンス用に `#[allow(dead_code)]` で残置
- `graph_width()` も `#[allow(dead_code)]` で残置
- `lanes_for_width(width_px)` 関数を新設（mod.rs 側で使用）

### mod.rs の変更点

- `DividerKind::BadgeCol`, `DividerKind::GraphCol` を追加
- `KagiApp` に `badge_col_w: f32`, `graph_col_w: f32` フィールドを追加
- `from_snapshot`, `with_error` でデフォルト値を設定
- `divider_drag_move` リスナーに `BadgeCol`/`GraphCol` ケースを追加（絶対座標方式）
- `render` 内に `col_header` (20px, flex_shrink_0) を WIP 行の上に追加
  - 「BRANCH / TAG」「GRAPH」「MESSAGE」ラベル + 2本のドラッグハンドル
- WIP 行のバッジ列幅を `badge_col_w` に変更
- WIP 行にグラフ列プレースホルダーと内側 divider スペーサーを追加
- `render_rows` に `badge_col_w`, `graph_col_w` 引数を追加
- `render_badges_column` に `badge_col_w: f32` 引数を追加し、`w(px(badge_col_w))` を使用
- `graph_canvas` 呼び出しに `visible_lanes` を渡す
- グラフ列は常に `w(px(graph_col_w))` の固定幅で描画（overflow_hidden で右端クリップ）
