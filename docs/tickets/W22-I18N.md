## 実装メモ(done)

### 追加 module: `src/ui/i18n.rs`(依存ゼロ)
- `Lang { En, Ja }` を `AtomicUsize`(0=En, 1=Ja)で保持 — `theme::ACTIVE` と同型。
  `lang()` / `set_lang()`(set_lang は settings.json `"lang"` へ永続化)/
  `init_lang()`(起動時解決 + `[kagi] lang: <slug>` ログ)。
- `Msg` enum(**69 キー**)+ `Msg::t(self) -> &'static str` を `match (lang(), self)`
  で網羅 match。翻訳漏れはコンパイルエラー。ドメインワード(Pull/Push/Branch/
  Stash/Pop/Undo/amend/checkout/cherry-pick/revert/discard/worktree/HEAD/branch/
  upstream 等)は両 arm に英語 verbatim で残置。
- 引数つき文は i18n 側ヘルパ fn(**2 本**): `wip_row_note(n)`(WIP 行注記)/
  `unstaged_not_included(n)`(commit panel の unstaged 警告)。複数形判定も chars 安全。
- 言語解決の優先度: `KAGI_LANG=en|ja` → settings.json `"lang"` → `LANG`/`LC_ALL` が
  `ja` 始まりなら Ja → 既定 En。`resolve_lang()` は副作用なし(テスト容易)。

### 永続化
- `theme.rs` の `SETTINGS_KEYS` に `"lang"` を追加(write_setting の round-trip 対象)。
  `set_lang` は theme と同じ手書き JSON 機構で `"lang"` を upsert(他キー非破壊)。

### メニュー配線(theme submenu と同型)
- `commands.rs`: `LangEnglish` / `LangJapanese` action、COMMANDS 2 行、`command_state`
  で常時 Enabled、`lang_submenu()`(✓ マーカー付き)を View メニューへ追加、
  `handle_menu_command` の `lang.*` arm、`KagiApp::set_lang`(set_menus 再構築 + notify)。
- `mod.rs`: `register_menu_actions` に `LangEnglish`/`LangJapanese` を登録。
- `main.rs`: `theme::init_active()` 直後に `i18n::init_lang()` を追加。

### Msg 化した文字列(src/ui/ のみ。**68 箇所の `.t()` 呼び出し + 2 ヘルパ**)
- 既存の日本語ハードコード全部: `別の操作が実行中です`(×14)、Busy footer
  `… 実行中…`(×12)、`…: 開始しました` トースト(×12)、commands.rs の Disabled 理由
  (×14)、context_menu.rs の disabled 理由(×11)、Pull/Push/Pop/Undo ツールバーガード、
  checkout/compare/amend/reset の説明文、WIP 行注記、unstaged 警告、`既に HEAD です`、
  dirty stash recovery 文、`No operations yet` / `No local branches` 空状態、
  `Refreshed` / `Opened in Finder` トースト 等。
- **非対象は据え置き**: smart_commit.rs の ADR-0044 同意 4 文言(verbatim 英文、未変更)、
  ドメインワード・単語ボタン・列ヘッダ・SHA・branch 名、内部 eprintln デバッグログ、
  smart-commit 出力言語トグル `Lang: EN/日本語`(message_gen 側の別概念)。

### 逸脱
- `tests/i18n_test.rs` は当初想定の「`Msg::t` 直接 unit test」ではなく、i18n が bin crate
  所属で integration harness から到達不可のため、**バイナリ spawn による env 解決の
  ブラックボックステスト**とした。`Msg::t` の言語切替 unit test は `i18n.rs` 内に同等以上を実装済み。
- `command_state` の内部防御 fallback `Disabled("unknown command")` は UI に出ない開発用
  文言のため英語リテラルのまま残置(翻訳対象外)。
